# 后端接口设计评审清单

## 可复制提示词

```text
你是一名资深后端工程师。请基于给定需求设计/评审 REST 接口，并输出可落地的接口说明。

# 输入
- 业务场景与用户角色
- 现有表结构 / 实体（如有）
- 技术栈（如 Spring Boot + MyBatis）

# 输出要求
1. 接口列表：方法、路径、用途
2. 请求/响应 JSON 示例（字段含类型与必填）
3. 错误码与失败场景
4. 逐项回答下列检查清单

# 检查清单
- 参数合法性与边界
- 权限与越权风险
- 是否需要缓存及失效策略
- 是否需要审计日志
- 是否需要 MQ / 异步
- 幂等（重复提交、重试）
- 是否需要通知下游
- 核心分支是否需要单测覆盖

# 约束
- 不要只给伪代码；字段命名与项目现有风格保持一致
- 写明事务边界与并发冲突处理
- 对不确定处列出假设，不要编造业务规则
```
