# 日程创建抽屉 · 忙碌时间轴组件提示词

创建/编辑日程时，左侧用 DatePicker 精调时间，右侧用忙碌时间轴可视化拖选时段，并叠加参与人已有日程。下面是一份整理后的通用提示词，可直接丢给 AI 实现。

## 使用方式

1. 先贴项目技术栈（如 Vue3 + TS + Ant Design Vue + 现有抽屉组件路径）。
2. 再贴「通用提示词」全文。
3. 最后补充你们仓库里的真实 API、字段名、目录约定。

## 通用提示词（可复制）

```text
你是一名熟悉 Vue3 + TypeScript 的前端工程师。请在「日程创建/编辑抽屉」中实现 PC 端「忙碌时间轴」组件，并与左侧表单双向同步。

# 目标
用户除了用 DatePicker 精确输入起止时间外，还能在右侧时间轴上拖拽蓝色选中框的上下边界调整时段；同时看到参与人忙碌色块，便于避开冲突。

# 布局与端差异
- PC：抽屉双栏。左表单（主题、归属人、起止时间 DatePicker、参与人、会议室等），右时间轴。
- 移动端：不展示时间轴；时间仅用表单 DatePicker。可选：点击忙碌头像弹窗查看时段。

# 时间轴交互（核心）
1. 纵轴为锚定日当天的小时刻度，每小时固定像素高度（默认 48px，可配置）。
2. 蓝色「当前选中时段」矩形覆盖 startTime ~ endTime。
3. 选中框上下各有拖拽手柄：
   - 拖上边 → 改 startTime
   - 拖下边 → 改 endTime
4. 将指针相对轨道的 Y 转为时间；按 15 分钟吸附（步长可配置）。
5. 约束：start < end，且至少间隔一个吸附步长。
6. 结束时间允许拖出当天底部，支持跨天。
7. 顶部提供：前一天 / 今天 / 后一天，切换 anchorDate；选中时段变化时尽量滚动到可见区域。
8. 提示文案：「拖动蓝色框上下边界可调整起止时间；点击图例可筛选单人忙碌」。

# 数据绑定
- 时间轴与表单 startTime、endTime 双向绑定（v-model）。
- 另有 anchorDate（锚定展示日），尽量与 startTime 所在日同步。
- 组件内统一使用 `YYYY-MM-DD HH:mm:ss`；提交后端前再转毫秒时间戳（或按现有 API 约定转换）。

# 忙碌展示
1. 参与人或可见时间范围变化时，防抖请求「参与人忙碌」接口。
2. 时间轴叠加半透明色块：每人一色，展示姓名 + 时段 + 主题；无权限时显示「忙碌」。
3. 支持「仅显示忙碌」、点击图例聚焦某人。
4. 忙碌色块只读，不可点选改时间；改时间只通过蓝色选中框。
5. 编辑已有日程时，查询忙碌需传 excludeEventId，避免把自己算进冲突。

# 组件拆分建议
- ScheduleDrawer：抽屉壳 + 左右布局
- ScheduleForm：左侧表单
- BusyTimeline：右侧时间轴（刻度、选中框、忙碌层、日期切换、图例）
- 抽离纯函数：yToTime / timeToY / snapToStep / clampRange / clipBusyToDay

# 非功能要求
- 同时兼容 mouse 与 touch。
- 兼容后端时间多种格式：毫秒、字符串、LocalDateTime 数组。
- 全天/跨天忙碌块要正确裁剪到当前可见轨道。
- Vue3 + TS + `<script setup>`；职责清晰；关键拖拽与坐标换算处写简要注释。
- 优先复用项目现有组件库（如 Ant Design Vue）与日期工具，不要引入无关重依赖。

# 验收标准
- 拖蓝框上下边，左侧 DatePicker 同步变化。
- 改 DatePicker，蓝框位置同步变化。
- 15 分钟吸附正确，起止不会交叉。
- 有参与人时能看到忙碌色块，可筛选单人。
- 跨天结束时间可用；切换日期后选中框与滚动行为正常。
- 移动端不渲染时间轴，表单仍可完整创建/编辑。

请先给出组件树、关键 props/emits、坐标换算伪代码，再实现可运行代码；不要一次性糊成单文件巨型组件。
```

## 写提示词时的优化点

相对随手需求描述，这份提示词主要做了这些收紧：

| 维度 | 做法 |
|------|------|
| 角色与边界 | 写明技术栈、交付形态、禁止巨型单文件 |
| 交互规则 | 把吸附、最小时长、跨天、滚动写成硬约束 |
| 数据约定 | 统一时间字符串格式，提交时再转时间戳 |
| 组件职责 | 要求拆分 Timeline / 换算函数，方便评审与单测 |
| 验收清单 | 用可勾选行为代替模糊「体验要好」 |

## 落地时记得补的上下文

把下面信息贴在提示词后面，命中率会高很多：

- 现有抽屉/表单文件路径与字段名
- 忙碌接口 URL、请求参数、响应结构示例
- `excludeEventId` 字段名与权限字段含义
- 设计稿或截图（双栏比例、色板、字号）
- 是否已有 dayjs / date-fns 等工具库

## 小结

提示词的价值不在「写得长」，而在把**交互约束、数据契约、端差异、验收标准**一次说清。日程时间轴这类强交互组件，尤其适合用「规则清单 + 验收清单」驱动 AI 实现。
