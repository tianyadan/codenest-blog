## 答案

## 简要回答

Java 常见的垃圾回收算法包括标记-**清除**、**复制**、**标记-整理**和**分代收集**。现代 JVM 垃圾回收器通常会组合多种算法使用，以兼顾吞吐量、停顿时间和内存利用率。

## 专业名词解释

### 垃圾回收（GC）

垃圾回收是 JVM 自动识别并释放不再被程序使用对象所占内存的过程，避免开发者手动管理内存。

### 分代收集

根据对象存活时间长短将堆内存**划分为新生代和老年代**，**不同区域采用不同回收算法**，以提高回收效率。

## 面试回答

Java 中常见的垃圾回收算法主要**有四种**。

第一是**标记-清除算法**，先标记存活对象，再清理未被引用的对象。优点是实现简单，但会产生大量内存碎片。

第二是**复制算法**，将内存划分为两块，每次只使用其中一块，回收时把存活对象复制到另一块。优点是没有内存碎片，缺点是内存利用率较低，**常用于新生代**。

第三是**标记-整理算法**，先标记存活对象，再将存活对象向一端移动，最后清理边界外的内存。它解决了内存碎片问题，**常用于老年代**。

第四是**分代收集算法**，它不是一种具体算法，而是一种**设计思想**。根据对象生命周期特点，新生代通常采用复制算法，老年代通常采用标记-清除或标记-整理算法。现代 JVM 的 `G1`、`ZGC`、`Parallel GC` 等垃圾回收器**本质上都是基于分代思想并结合多种回收算法实现的**。

## 核心要点

- **标记-清除**：会产生内存碎片
- **复制算法**：空间换时间
- **标记-整理**：解决碎片问题
- **新生代**：常用复制算法
- **老年代**：常用整理类算法
- **分代收集**：是主流设计思想

## 面试追问

### 1. 为什么新生代适合使用复制算法？

新生代中的对象大多数朝生夕灭，每次 GC 后存活对象数量较少，因此复制成本较低。通过**直接复制存活对象**，可以快速完成回收并避免内存碎片。

### 2. 为什么老年代不适合直接使用复制算法？

老年代对象存活率较高，如果采用复制算法，需要移动大量对象，并且需要预留较大空闲空间，内存开销和复制成本都比较高，因此通常**采用标记-整理算法**。
