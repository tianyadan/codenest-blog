## 答案

自旋（spin）：是行为，原地等待，不让出 CPU，循环检查某个条件是否改变。线程不挂起、不睡觉，让 CPU 原地循环等待。

经典示例：

```Java
while (lock_is_held) {
    // 什么都不干，就在这瞎转，等锁可能马上释放
}
```

可以把它想象成： 我不排队，也不坐下，就在你背后站着等你用完。

这种方式**很消耗 CPU**，但能换来比**阻塞/唤醒**更快的响应。（因为上下文切换太贵了）

所以 JVM 会在某些情况下短暂自旋，比如重量级锁竞争。
