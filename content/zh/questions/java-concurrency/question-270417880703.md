## 答案

ConcurrentHashMap 是 Java 提供的线程安全 HashMap，通过 **CAS** + **synchronized** 实现高并发控制，相比 **Hashtable** 具有更好的性能。

ConcurrentHashMap 特点：

- 线程安全
- 高并发性能（重点）

它解决了普通 HashMap **多线程下不安全（甚至会死循环）的问题**。其核心思想是 “**分段+CAS+局部锁**”。

在 JDK 1.7 时使用分段锁（Segment），类似把 Map 拆成多个“小 Map”，每个 Segment 一把锁 ，多线程可以同时操作不同的 Segment。本质是减少锁冲突。 

在 JDK1.8 （现在主流）已经彻底升级。核心结构： **数组 + 链表 + 红黑树（和 HashMap 一样）** 但并发控制已经改变 ：

1 ） CAS （无锁操作）：插入空位置时，直接 CAS，不加锁，速度极快 （因为 CAS 是 CPU 原子指令）。
2 ） synchronized （锁链表/桶）：**只锁当前桶（bin），不锁整个表，粒度更细**。
3 ） volatile ： 保证可见性（最新数据）


**扩展知识：**

ConcurrentHashMap 为什么快？


| 操作 | 机制 | 是否加锁 |
| --- | --- | --- |
| 读(get) | volatile | 无锁 | 
| 写(put) | CAS+Synchronized |  有锁，但粒度很小 |
| 扩容 | 多线程协助扩容 | 无锁 |
