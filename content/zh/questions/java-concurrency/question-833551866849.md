## 答案

ReentrantLock 其实就是基于 AQS 实现的一个可重入锁，支持公平和非公平两种方式。

内部实现依靠一个 state 变量和两个队列：同步队列和等待队列。

利用 CAS 修改 state 来争强锁。

争抢不到则入同步队列等待，同步队列是一个双向链表。

条件condition 不满足时候则入等待队列等待，是个单向链表。

是否公平锁的区别在于：线程获取锁时是加入到同步队列尾部还是直接利用 CAS 争抢锁。
