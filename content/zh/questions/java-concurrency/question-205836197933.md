## 答案

重入锁指的是同一个线程在持有某个锁的时候，可以再次获取该锁而不会发生死锁。

示例代码：

```Java
public class ReentrantLockExample {
    private final ReentrantLock lock = new ReentrantLock();

    public void outer() {
        lock.lock(); // 上锁
        try {
            inner(); // 调用inner方法
        } finally {
            lock.unlock(); // 释放锁
        }
    }

    public void inner() {
        lock.lock(); // 上锁
        try {
            // critical section
        } finally {
            lock.unlock(); // 释放锁
        }
    }
}
```

- ✅ 这段代码干了啥？

类里有一个 ReentrantLock，然后：
	•	outer() 先 lock.lock() 上锁
	•	里面又调用 inner()
	•	inner() 也再 lock.lock() 上锁一次

也就是说：同一个线程连续两次获取同一把锁。


- ✅ 重点：为什么不会死锁？

如果用的是 synchronized 或者普通互斥锁，如果同一个线程锁住了某个资源，再次尝试获取同一把锁，很可能会死锁（自己堵自己）。

但 ReentrantLock 是可重入的：

同一个线程可以多次获取同一把锁，不会被自己挡住。

**而且内部会维护一个计数器：**
	•	第一次 lock() → count = 1
	•	第二次 lock() → count = 2
	•	只有当 unlock() 也调用两次，count 才会变回 0，锁才真正释放。
        
所以代码执行过程：
outer() lock ✅
inner() lock ✅
inner() unlock ✅
outer() unlock ✅
锁最终释放 ✅


- ✅ 这段代码其实说明了两个核心点

1️⃣ ReentrantLock 允许重入

同一个线程 调用 lock() 多次是合法的，不会死锁。

2️⃣ unlock() 必须成对出现

你加几次锁，就得解几次锁，否则锁永远不会释放，后续线程会被堵死。

这点比 synchronized 更“危险”，因为你必须自己管理解锁次数。
