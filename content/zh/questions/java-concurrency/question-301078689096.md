## 答案

**答：**
 
**1) volatile**

volatile 会禁止 **某些** 类型的指令重排序，并添加内存屏障，保证：

- 写 volatile：写 ->  刷新到主内存 ->  禁止后面的读写重排

- 读 volatile: 读前添加屏障 -> 禁止前面的读写重排

所以 DCL 要这样写才安全：

```Java
private static volatile Singleton instance;
```

**2) synchronized**

 天然带内存屏障，进入/退出锁都会触发屏障
  
**3) 显示内存屏障(Unsafe / VarHandle )**

Java 9 后用 VarHandle 已经更高级了。
