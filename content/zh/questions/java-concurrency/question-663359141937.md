## 答案

在 Java 中 ，volatile 和 synchronized **是两种用于实现线程安全的关键字**，但他们的作用和使用场景有很大区别。

1 ） **volatile** 

- 只适用于字段变量。
- 确保变量的可见性。就是每次读取到volatile 变量时，都会从主内存中读取最新的值，而不是从线程的本地缓存中读取。写入 volatile 变量时，会立即将其刷新到主内存。
- 不保证操作的原子性，如 i++这样的复合操作，就算用 volatile 声明也可能有线程安全问题。
- 性能开销小，因为不加锁，只保证线程间的可见。



2 ）**Synchronized**

- 用于方法或代码块
- 可以保证原子性。通过锁来确保同一时间只能有一个线程执行被保护的代码块。
- 可以保证可见性。因为加锁会从主存读最新值，放锁会将数据写入主存。
- 性能开销比较大，因为需要获取和释放锁，会导致线程阻塞。
