## 答案

 **一句话概括：AQS 是一个用来构建锁、同步器的底层框架，它用一个 FIFO 队列+一个状态标识来管理线程的获取与释放。**

AQS（AbstractQueuedSynchronizer） 是 Java 并发框架的底层灵魂，几乎所有耳熟能详的同步器都离不开它，比如：

- ReentrantLock
- Semaphore
- CountDownLatch
- ReentantReadWriteLock
- FutureTask

 **如果把 Java 并发比喻成一个城市的交通系统，AQS 就是交通信号灯、路障、路口调度“统一调度中心”。**
 
 它做的事情包括：
 
 - 管理“谁能拿锁”
 - 决定“谁应该等待”
 - 决定“谁应该背唤醒”
 - 保证这一切都是安全的（CAS+volatile）

**核心组成：**

1. **state**（关键状态变量）

一个 volatile int state ，用来表示锁的状态：

- 对于 ReentrantLock: state = 获取锁的次数

- 对于 CountDownLatch: state = 计数

- 对于 Semaphore: State  =  可用许可数量。

state 并没有固定含义，全看怎么继承 AQS 来使用。

2. **CLH 双向队列**（等待队列）

当一个线程抢锁失败，就被 AQS“塞进”一个 FIFO 队列里，以 Node 形式存在。

```Java
head <-> node1 <-> node2 <-> node3
```
线程在队列里“排队”等通知。
