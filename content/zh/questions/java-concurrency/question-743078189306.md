## 答案

**双重检查锁定**(Double-checked Locking,**DCL**) 是多线程领域一个非常经典、也非常容易写错的的单例(Singleton)创造模式，
**核心目标很简单：在多线程环境下，用尽可能减少锁的开销，安全地创建一个单例对象。**

为什么要“双重检查”检查？

因为:

1 ） 不加锁的情况下检查一次 ： **快**

2 ） 加锁之后再检查一次 ：**安全**

**示例代码：**
```Java

public class Singleton {

    // 使用 volatile 非常关键！！！
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                 // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {         // 第二次检查
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

```

**⚠️ 为什么必须加 volatile？**

没有 volatile 会出现著名的**指令重排**问题：

new Singleton() **不是原子操作**，实际上会被拆成三步：
	1.	分配内存
	2.	初始化对象
	3.	把实例指向内存地址

指令重排可能变成：1 → 3 → 2

导致：
	•	线程 A 执行 1、3，这时 instance 已经“不再是 null”
	•	线程 B 进来看到 instance 不为 null，就直接返回
	•	但此时对象其实还没初始化完 → 出问题！

volatile 禁止重排，确保安全。

✨ **总结**

	•	DCL 是为了在多线程环境下实现高性能的懒加载单例。
	•	通过两次检查避免不必要的加锁，同时保证线程安全。
	•	volatile 是 DCL 最关键的修复措施，用于避免指令重排导致的半初始化对象。
