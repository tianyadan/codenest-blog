## 答案

在 Java 中控制多个线程的执行顺序有很多种方法：

1）CompletableFuture，它内部用thenRun 的方法，假设我们现在有三个 T1、T2、T3 需要按需执行，那么仅需使用以下伪代码：


```Java
CompletableFuture.runAsync(() -> {do t1 sth})
   .thenRun(()-> {do t2 sth})
   .thenRun(()-> {do t3 sth});
```

2）synchronized+wait()/notify()，通过对象锁和线程间通信机制来控制线程执行顺序。

3）ReentrantLock+condition。

4）Thread 类 join()， 通过调用这个方法，可以使一个线程等待另一个线程执行完毕后再继续执行。

5）CountDownLatch，使一个或多个线程等待其他线程完成各自工作后再继续执行。

6）CyclicBarrier，使多个线程互相等待，直到所有线程都到达某个共同点后再继续执行。

7）Semaphore，控制线程的执行顺序，适用于需要限制同时访问资源的线程数量的场景。

8）线程池，内部仅设置一个线程来执行任务，按顺序将任务提交到线程池中就可以了。
