## 答案

### **一句话总结：**

指令重排序就是编译器、JIT、CPU 为了优化程序执行速度，**在不违反“单线程语义的前提下”，偷偷调整指令执行顺序。** 但是如果在多线程中，指令重排可能导致线程之前的操作出现不同步或不可见的现象，因此 Java 提供了内存模型（JMM）和相关机制（如 volatile 和 Synchronized ）来限制这种行为，确保并发操作执行正确性。

### **补充：**

#### **1）为什么会发生重排序？**

**答：为了性能。**

- 编译器优化：把代码调整执行顺序，提高运行效率。

- 处理器流水线优化：CPU 会并行、乱序执行某些指令。

- 内存系统优化：写缓冲、缓存一致性协议也会导致“看起来像是乱序秩序”。


**2）JVM 将重排序分为三类**

 1. 编译器优化的重排序。
 2. 处理器优化的重排序。
 3. 内存系统导致的重排序

并且这些都不能控制，但是能通过内存屏障避免。
