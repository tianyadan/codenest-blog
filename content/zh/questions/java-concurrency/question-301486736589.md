## 答案

理论上线程池的多少受到系统资源的影响，尽量不要随意开线程池，最致命的问题就是 CPU 竞争，买哪个线程池都有自己的线程，总线程数远超过 CPU 核心数。

结果就会导致：
- 线程互相抢 CPU
-  上下文切换暴涨
-  性能反而下降
-  内存爆炸，每个线程≈1MB 栈内存 10 个线程池*20 个线程 =  200MB+ 很容易 OOM



 推荐线程池规范设计
 
 1 ） IO 密集型线程池
 
 
```java
ExecutorService ioPool = new ThreadPoolExecutor(

    10, 50,

    60, TimeUnit.SECONDS,

    new ArrayBlockingQueue<>(200)

);
```
 可用于 调用 AI、调 Http、查数据库。
 
 2 ）CPU 密集型线程池
 
 ```java
 int cpu = Runtime.getRuntime().availableProcessors();
 
 ExecutorService cpuPool = new ThreadPoolExecutor (
 cpu,cpu,
 0,TimeUnit.SECONDS,
 new LinkedBlockingQueue<>();
 
 ```
 可用于 JSON 处理，数据计算，排序
 
  3 ）关键任务线程池（隔离）
  
  ```java
   ExecutorService criticalPool = new ThreadPoolExecutor(
   2,5,60,TimeUnit.SECONDS,new ArrayBlockingQueue<>(50);
   ```
   
   可用于 PDF 导出、报告生成 ... 防止被普通请求拖死
   
   
**总结：**

> 在实际项目中，线程池的数量没有硬性限制，但不能随意创建过多线程池，否则会导致 CPU 竞争、内存浪费以及调优困难。
一般会按照任务类型划分线程池，比如 IO 密集型、CPU 密集型以及关键业务隔离线程池，通常控制在 3 到 5 个以内。
同时会根据任务特点合理设置核心线程数、队列大小和拒绝策略，从而保证系统的稳定性。
