## 答案

核心作用在于：**任务异步化**（不阻塞主线程）、**任务并行化**（提高吞吐量）、**资源控制**（编创建过多线程）

常见使用场景：

1. web 服务器中的请求处理：每个 Http 请求都需要分配线程处理，例如 Tomcat等内部都用了线程池，请求时进入分配工作线程；处理完自动回收。

2. 异步任务执行 ：当主线程不想阻塞时，可以用线程池异步执行任务，例如：用户注册后异步发送邮件，上传文件后异步计算缩略图，下单后异步生成发票和日志记录。

3. 批量数据处理/并行计算 ： 比如有一个需要处理 10 万条数据的任务，可以用线程池拆分多段并行处理：
```Java
   for (List<SubList> batch : splitList(bigList :1000)){
    executor.submit(()-> processBatch(batch));
    } 
```
 充分利用多核 CPU，提高吞吐量。
 
 4. 消息队列消费者
 线程池非常适合多线程消费消息队列中的任务：
 - 每条消息独立消费；
 - 线程池统一调度；
 - 可根据系统负载动态调整核心线程数。

常见于：RabbitMQ / Kafka 消费者。


5. 定时任务/延时任务

使用 ScheduledThreadPoolExecutor 可以实现周期性任务：

```Java
ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(2);
scheduler.scheduleAtFixedRate(() -> checkHealth(),0,10,TimeUnit.SECONDS);
```

6. 并发 I/O 操作

比如多个文件下载、批量爬虫、调用多个外部 API 接口

executor.invokeAll(task);

7.写日志、后台任务

 logExecutor.execute(()->saveLogToDB(log));
