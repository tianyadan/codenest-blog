## 答案

Redis 提供了**五种基本数据类型**。 (String 、List 、Set、Zset、Hash ) 和**四种高级数据类型** （BitMap、Geo、HyperLogLog、Stream）每种类型都有其特定的使用场景和底层实现机制。

**基本数据类型：**

String：
最基本的类型，可以存储文本、数字或二进制数据，最大容量为512MB。
典型应用场景包括缓存用户会话、页面数据，以及作为计数器（如文章阅读量、点赞数），支持原子递增操作（INCR），底层实现是采用 Redis 自己封装的 SDS (Simple Dynamic String -简单动态字符串 ，因为 C 字符串太“原始”，不安全、不高效、不适合高并发场景。Redis 需要一个更“工程化”的字符串实现。)

Hash
键值对集合，适合存储对象属性。
例如存储商品详情时，商品ID作为key，价格、库存、名称等字段存储在同一个Hash中，修改单个字段无需整体覆盖。

List
有序的字符串列表，底层实现为**双向链表**，**支持两端操作**。
常用作消息队列，通过**LPUSH**生产消息，**RPOP**消费消息，实现简单的生产者消费者模式。

Set
无序且元素**不重复的集合，查找和去重效率高**。
适用于标签系统、记录独立访客、共同关注等需要去重或集合运算的场景。

Zset
类似于Set，但每个元素附带一个score用于排序，底层实现为**跳表**。
典型应用包括**排行榜**（如游戏积分榜、热搜榜），支持按score排序并获取Top N。

**高级数据类型：**

BitMap
引入于Redis 2.2，使用位存储数据，每个bit仅表示0或1，空间利用率极高。
适用于统计场景，如**用户签到**情况（每个用户仅占1bit），支持SETBIT设置状态和GETBIT读取状态。

HyperLogLog
引入于Redis 2.8，概率性数据结构，用于估算基数，固定占用12KB内存，误差率约0.81%。
适用于大规模数据统计，如网站UV（独立用户数），支持PFADD记录用户和PFCOUNT估算用户数。

GEO
引入于Redis 3.2，**存储地理位置信息**，支持经纬度存储和空间查询，底层实现为Zset。
典型应用包括“附近的人”、外卖配送距离计算，支持GEOADD存储位置和GEORADIUS查询范围内对象。

Stream
引入于Redis 5.0，专为**消息队列设计**，支持全局唯一消息ID和消费组模式。
相比List和Pub/Sub，优势在于消息持久化和消费者重启后继续消费，适用于轻量级消息传递场景。


**数据类型与底层编码关系**

每种数据类型在底层可能有多种编码实现，根据数据量和元素大小自动切换。
例如，Hash在小数据量时使用listpack编码，大数据量时切换为hashtable编码，切换阈值可通过配置调整。
