## 答案


| 数据类型 | 小数据量编码 | 大数据量编码 | 切换阈值 |
| --- | --- | --- | --- |
| String | int/embstr | raw | 44 字节 |
| List | quicklist | quicklist| ~ |
| Hash | listpack | hashtable | 512 字段或单值超过 64 字节|
| Set  | intset/listpack | hashtable | 512 个元素或含非整数 |
| Zset  | listpack  | skiplist+hashtable | 128 个元素或单值超过 64 字节 |

这些阈值可以通过配置调整，比如 `hash-max-listpack-entries` 控制Hash 转换的元素数阈值。

**扩展知识 ：**

- listpack 连续内存，节省空间但修改成本高
 
- hashtable 分散内存，占空间但操作快

- quicklist = 链表 + 压缩数组（listpack）的组合体

```txt
quicklist (一排火车（链表） + 每节车厢坐很多人（listpack）
   ↓
双向链表（外层）
   ↓
每个节点里放一个 listpack（内层）
```
**快速记忆：**

Hash / Set：512 + 64
Zset：128 + 64
String：44
