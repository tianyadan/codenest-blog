## 答案

## 简要回答

MSET、MGET 是 Redis 提供的单条批量命令，只处理同类字符串读写；`Pipeline` 是客户端批量发送多条命令的通信优化机制，可组合不同命令，但默认不提供原子性保证。

## 专业名词解释

### Pipeline

`Pipeline` 是客户端将多条 Redis 命令连续发送到服务端，而不逐条等待响应，再统一读取结果的机制。它主要减少网络往返次数，提高批量操作吞吐量。

### 原子性

原子性表示一组操作在外部看来要么整体完成，要么不会暴露中间状态。Redis 单条命令执行具有原子性，但普通 `Pipeline` 中的多条命令并不是一个原子操作。

## 面试回答

`MSET`、`MGET` 和 `Pipeline` 都能降低批量访问时的网络开销，但它们的层次和语义不同。

`MSET`、`MGET` 是 Redis 服务端原生命令。`MSET` 一次设置多个字符串键值对，并且整条命令具有原子性，其他客户端不会看到只更新了部分 key 的中间状态；`MGET` 一次读取多个字符串 key，并按输入 key 的顺序返回对应结果。它们适用于操作类型一致、数据结构都是字符串的简单批量读写。

```bash
MSET user:1:name Tom user:2:name Jerry
MGET user:1:name user:2:name
```

`Pipeline` 则不是新的数据操作命令，而是客户端侧的发送方式。客户端可以将多条不同类型的命令批量发送，例如同时发送 `SET`、`HGET`、`INCR`、`EXPIRE`，服务端仍然逐条执行并逐条返回结果，只是减少了客户端和服务端之间反复等待响应的网络往返。

```bash
SET order:1:status created
HGET user:1 profile
INCR metrics:order:count
EXPIRE order:1:status 3600
```

关键区别在于原子性：`MSET` 本身是单条原子命令；普通 `Pipeline` 只是批量传输，多条命令之间可能穿插执行其他客户端的命令，不能用于保证业务一致性。若要求多条命令整体不被插入，需要结合事务 `MULTI`/`EXEC`，或者使用 Lua 脚本、具备对应语义的原生命令。

选择上，同类字符串批量读写优先考虑 `MSET`、`MGET`，语义更直接；需要批量执行不同命令，或批量设置过期时间、操作不同数据结构时，使用 `Pipeline` 更灵活。但无论采用哪种方式，批次都不应无限增大，否则会导致请求包、响应结果和服务端执行时间过大，影响延迟与稳定性。

## 核心要点

- `MSET`、`MGET` 是服务端原生命令
- `Pipeline` 是客户端批量传输机制
- `MSET` 可原子地写入多个字符串 key
- 普通 `Pipeline` 默认不保证原子性
- `Pipeline` 能混合发送不同命令
- 批量大小过大仍可能造成阻塞

## 面试追问

### 1. MSET 和多条 SET 放入 Pipeline，效果完全一样吗？

不一样。两者都能减少网络往返，但 `MSET` 是一条原子命令，多个键不会暴露部分更新状态；多条 `SET` 放入普通 `Pipeline` 后仍是多条独立命令，其他客户端可能在这些命令之间执行操作，因此不能替代 `MSET` 的原子语义。

### 2. Pipeline 与 Redis 事务 MULTI/EXEC 有什么区别？

`Pipeline` 关注性能，目标是批量发送命令、减少 RTT；`MULTI`/`EXEC` 关注执行隔离，事务队列中的命令在执行阶段不会被其他客户端命令插入。两者可以结合使用，但普通 `Pipeline` 本身不等于事务，也不保证多命令原子执行。

### 3. 在 Redis Cluster 中使用批量命令需要注意什么？

`MGET`、`MSET` 这类涉及多个 key 的原生命令通常要求相关 key 位于同一个 hash slot，否则无法直接在单个节点执行。实践中可通过 hash tag，例如 `user:{1001}:name`、`user:{1001}:age`，让需要共同操作的 key 落到同一 slot；跨 slot 批量访问则通常由客户端拆分处理。

## 常见误区

**误区：** 使用 `Pipeline` 批量发送命令后，这些命令就会像事务一样原子执行。  
**正确理解：** `Pipeline` 主要优化网络通信，普通情况下不提供多条命令的原子性；一致性要求应依赖原生命令、事务或 Lua 脚本。

**误区：** `MSET`、`MGET` 可以替代所有批量操作需求。  
**正确理解：** 它们只适合字符串类型的同类批量读写；涉及不同命令、不同数据结构或附加操作时，`Pipeline` 更灵活。
