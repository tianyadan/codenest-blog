## 答案

Redis 里的跳表（skip list）本质上就是一种“**用空间换时间**”的有序链表优化结构，说白了就是：在普通链表上加多层“高速通道”，让查找不再傻傻一层一层走。

**一、跳表核心思想**：

普通有序链表是这样的：

`1 → 3 → 5 → 7 → 9 → 11`

你要查 9，只能从头一个个比，**时间复杂度 O(n)**。

跳表的做法是：在上面加“索引层” 

```txt
Level 2:       1 ------ 5 ------ 9
Level 1:    1 --- 3 --- 5 --- 7 --- 9 --- 11
```
查找过程：

1.先从最高层开始
2.不断向右走，直到“快超过目标”
3.然后下降一层继续

这个过程其实就像二分查找的感觉，**最终复杂度：O(log n)**

**二、Redis 跳表的结构设计（重点）**

Redis 的跳表实现比教科书稍微复杂一点，但核心一致。

节点结构（zskiplistNode）：


```c
typedef struct zskiplistNode {
    double score;        // 排序分数
    sds ele;             // 元素值
    struct zskiplistNode *backward; // 后退指针
    
    struct zskiplistLevel {
        struct zskiplistNode *forward; // 前进指针
        unsigned long span;            // 跨度
    } level[];
} zskiplistNode;
```

关键点：

- score：排序依据（Sorted Set 用）

- 多层 forward 指针：实现跳跃

- **span（跨度）： 这是 Redis 的精华，用来算排名**

跳表结构（zskiplist）


```c
typedef struct zskiplist {
    struct zskiplistNode *header, *tail;
    unsigned long length;
    int level;
} zskiplist;
```

**三、层级是怎么来的？（重点中的重点）**

跳表最核心的问题：一个节点有多少层？ 
Redis 用的是：**随机概率（P = 0.25）生成层数**

伪代码类似：

```c
int level = 1;
while (random() < 0.25) {
    level++;
}
```

效果是：
 
| 层级 | 节点数量比例 |
| --- | --- |
|  1 | 100% |
|  2 | 25% |
|  3 | 6.25% |
|  4 |  更少 |

**越高层越稀疏 = 天然索引结构**

**四、查找过程**

查找一个 score：
1.从最高层开始
2.当前节点向右走（forward）
3.如果下一个节点 score > 目标 → 下移一层
4.重复直到第 1 层
**时间复杂度：O(log n)**

**五、插入过程**

插入比查找多一步：

1.找到插入位置（每一层）
2.记录路径（update 数组）
3.随机生成层数
4.修改每一层的指针

Redis 还会维护：
•span（排名）
•backward（反向链表）
