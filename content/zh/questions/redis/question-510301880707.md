## 答案

Redis Pipeline（管道）本质上是：
**客户端一次性向 Redis 发送多条命令，减少网络往返次数，从而提高 Redis 吞吐量。**
它的核心目标：减少网络 IO 、提高批量操作性能 。
很重要的一点：Pipeline 不保证原子性。 它只是批量发送命令 ，**如果 Pipline 中有一条错误指令 Redis 会报错，但是继续执行**。

- 不保证原子性
- 不会回滚
- 不会因为某条失败后中断后续命令

**适合：批量写入 、批量查询 、缓存预热 、大批量数据处理**

**扩展知识 ：**

一、Redis 为什么需要 Pipeline？
先理解 Redis 最大性能损耗在哪里 -- 很多人以为Redis 慢在执行命令 ，其实不是，Redis 单条命令执行通常只有微秒级，真正慢的是网络通信 。如果10000 条命令RTT（网络往返）网络来回会非常恐怖。

二、Pipeline 是怎么优化的？
Pipeline 允许客户端一次性发送多条命令 ，Redis也会一次性返回结果 。

三、Pipeline 的本质？

注意Pipeline 不会减少 Redis 执行命令数量 ，它减少的是网络往返次数，所以Redis CPU 没变但吞吐量会暴增。

四、Pipeline 和事务有什么区别？（高频）

- Pipeline的目标在于提升性能 ，特点是批量发送命令、不保证原子性，中间可能被其他命令插队。

- Redis 事务则是保证命令连续执行，特点是保证执行期间不被插队，但不提升网络性能。 
总结一句话： pipline 解决性能问题，事务解决并发顺序问题。

五、 Pipline 适合哪些场景？
缓存批量写入、Redis 缓存预热、批量统计、导入大数据等
