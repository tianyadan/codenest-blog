## 答案

Redis 对于过期数据，并不是时间一到立刻删除。
因为如果 Redis 给每个 key 都单独开一个定时器：key 一多，CPU 会直接炸掉。所以 Redis 采用的是：**“惰性删除 + 定期删除”** 。

--- 
一、惰性删除（Lazy Expire）
核心思想：“你不访问，我就不删。”

比如：`user:token:1` 设置了：`EXPIRE 60`，60 秒后，这个 key 理论上已经过期了，但 Redis 不会主动立刻删除 ，而是当客户端再次访问这个 key 时：Redis 才会检查是否过期？如果过期立即删除返回 null 。

优点：
-  CPU 消耗低
-  不会主动大量扫描 key

缺点：
- 如果一直没人访问这个 key 会一直占内存 ，所以仅靠惰性删除不够。

---

二、定期删除（Periodic Expire）

Redis 会每隔一段时间随机抽查一批设置了过期时间的 key。如果发现key 已过期就直接删除。Redis 默认    100ms 左右执行一次，每次随机检查部分 key，不是全量扫描 （因为全量扫描数据量很大）

优点：
- 能清理长期没人访问的过期 key
- 避免内存泄漏

缺点：
- 不保证立即删除
- 有一定 CPU 消耗

---
三、为什么 Redis 不用“定时删除”？

所谓定时删除就是key 到期 -> 立刻删，听起来很完美，但问题是如果几百万 key同时过期，Redis CPU 会瞬间飙升，严重影响主线程性能，所以 Redis 放弃了这种方案。
