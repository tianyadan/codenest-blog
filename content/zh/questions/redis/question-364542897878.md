## 答案

## 简要回答

最简单可靠的方式是使用 Redis 的布隆过滤器能力，通过 `BF.RESERVE` 预设容量和误判率，再用 `BF.ADD`、`BF.EXISTS` 完成写入与判断。没有该能力时，也可基于 `Bitmap`、哈希函数和 `SETBIT`/`GETBIT` 自行实现。

## 专业名词解释

### 布隆过滤器

布隆过滤器是一种空间效率很高的概率型集合结构。它能判断某元素**一定不存在**或**可能存在**，但不能保证“存在”判断绝对准确。

### 误判率

误判率是指元素实际上不存在，但查询结果显示“可能存在”的概率。容量越大、哈希分布越合理，误判率通常越低。

### Bitmap

`Bitmap` 本质上是在 Redis 字符串上按位读写数据。每一位只保存 `0` 或 `1`，适合存储布隆过滤器中大量紧凑的标记位。

## 面试回答

使用 Redis 实现布隆过滤器，优先选择现成的布隆过滤器命令，而不是手写位图算法。创建时先根据预计元素数量和可接受误判率初始化，例如使用 `BF.RESERVE user:seen 0.001 1000000`，表示预计存储一百万个元素，误判率控制在千分之一。之后通过 `BF.ADD` 或批量命令 `BF.MADD` 添加元素，通过 `BF.EXISTS` 或 `BF.MEXISTS` 判断元素是否可能存在。

```bash
BF.RESERVE user:seen 0.001 1000000
BF.ADD user:seen user:1001
BF.MADD user:seen user:1002 user:1003
BF.EXISTS user:seen user:1001
BF.MEXISTS user:seen user:1002 user:9999
```

查询结果为 `0` 时，可以确定元素不存在；结果为 `1` 时，只能说明可能存在，因此典型应用是缓存穿透防护：过滤器判断不存在时直接拦截请求，判断可能存在时仍要查询缓存或数据库确认。

如果运行环境没有布隆过滤器命令，也可以自行实现：对一个业务键计算多个独立哈希值，将每个哈希值映射到固定长度位图的下标，写入时执行多次 `SETBIT key offset 1`，查询时执行多次 `GETBIT`；只要有一位为 `0`，元素一定不存在，全部为 `1` 才表示可能存在。多个位操作可通过 `BITFIELD` 或 Lua 脚本减少网络交互并保证一次判断过程的完整性。

工程上要注意三个问题：第一，创建过滤器前**应估算容量和误判率**，容量严重不足会使误判率明显升高；第二，删除数据库数据后**不能直接安全地删除普通布隆过滤器中的对应标记位**，否则可能影响其他元素；第三，**过滤器与数据库新增数据需要保持写入一致性**，否则可能把真实存在的数据误判为不存在。常见方案是先写数据库，再可靠地补写过滤器，并准备重建机制处理历史遗漏。

## 核心要点

- 优先使用 `BF.RESERVE` 创建过滤器
- `BF.EXISTS` 为 `0` 表示一定不存在
- 查询为 `1` 仍需访问真实数据源确认
- 无模块时可用 `Bitmap` 加多哈希实现
- 容量和误判率必须提前合理规划
- 新增数据漏写会产生严重业务错误

## 面试追问

### 1. 布隆过滤器为什么会误判，但通常不会把已加入元素判断为不存在？

多个元素可能将相同位图位置设置为 `1`，因此未加入的元素也可能碰巧命中全部位置，产生误判。对于已经正确加入的元素，其对应位置已被设置为 `1`，且普通布隆过滤器不会随意清零，所以正常查询不会得到不存在的结果。

### 2. 使用布隆过滤器解决缓存穿透时，数据新增和删除应如何处理？

新增数据时，**不能只写数据库而漏写过滤器**，**否则后续请求可能被错误拦截**。通常应在数据库写入成功后补写过滤器，并通过消息、重试或定期重建提升一致性。删除数据不必立即删除过滤器标记，因为误判只会带来一次无效查询，而错误清位可能影响其他元素。

### 3. 为什么更推荐使用现成命令，而不是自己基于 SETBIT 实现？

现成实现已经封装了哈希计算、容量管理、误判率控制和扩容策略，调用简单且更不容易出错。自行实现虽然便于理解原理，也适用于能力受限的环境，但需要自行解决哈希选择、位图长度、批量操作、并发访问和重建迁移等工程问题。

## 常见误区

**误区：** 布隆过滤器返回元素存在，就可以直接认为数据库中一定有该数据。  
**正确理解：** 布隆过滤器的“存在”**只是可能存在**，**仍需查询缓存或数据库确认**；它主要可靠地拦截明确不存在的数据。

**误区：** 布隆过滤器中的元素可以像 `Set` 一样直接删除。  
**正确理解：** 普通布隆过滤器的多个元素可能共享相同位，直接清除某个位会导致其他元素被误判为不存在。需要删除能力时，应考虑计数布隆过滤器或其他数据结构。
