## 答案

**一、基于内存操作（最根本原因）**
内存访问速度纳秒级，比磁盘快 1000 倍以上。
传统数据库（如 MySQL）需要：磁盘 IO、页加载、B+ 树查找

**二、单线程模型（避免线程切换开销）**
Redis 采用 单线程处理命令（核心网络模型是单线程）没有线程上下文切换，没有锁竞争，不需要加锁保证数据一致性

**三、IO 多路复用**
不需要为每个连接创建线程、减少系统调用、减少上下文切换

**四、数据结构高度优化**
Redis 不是简单的 KV。它底层做了大量优化：
- SDS（简单动态字符串）
- 压缩列表(ziplist)
- 跳表(skiplist)
- 哈希表渐进式rehash 

比如：Hash 小数据量时用ziplist 
有序集合就用 跳表?+hash 双结构

**五、命令是原子操作**
因为 Redis 是单线程执行命令的，所以：每个命令都是原子性的，不需要额外加锁。
