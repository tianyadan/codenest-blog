## 答案

## 简要回答

大量用户 UV 统计优先使用 Redis 的 `HyperLogLog`：用 `PFADD` 写入用户标识，用 `PFCOUNT` 获取近似去重数量，用 `PFMERGE` 合并多个时间窗口。它以极低内存换取少量统计误差，适合海量访问分析。

## 专业名词解释

### UV

UV，即 Unique Visitor，表示统计周期内去重后的访问用户数量。同一用户访问多次，通常只计为一个 UV。

### HyperLogLog

`HyperLogLog` 是一种基数估算结构，不保存完整用户集合，而是根据哈希分布估算去重数量。Redis 中每个 `HyperLogLog` 最多约占用 `12 KB` 内存，标准误差约为 `0.81%`。

### 基数估算

基数是集合中不同元素的数量。基数估算允许结果存在小幅误差，以显著降低海量去重统计所需的内存成本。

## 面试回答

统计大量用户唯一访问量时，如果业务允许近似结果，我会优先使用 Redis 的 `HyperLogLog`。每次用户访问时，以用户唯一标识作为元素写入按统计维度划分的 key，例如按天统计可以使用 `uv:page:1001:2026-05-27`，执行 `PFADD`；查询当天 UV 时执行 `PFCOUNT`。

```bash
PFADD uv:page:1001:2026-05-27 user:10001
PFADD uv:page:1001:2026-05-27 user:10002
PFADD uv:page:1001:2026-05-27 user:10001
PFCOUNT uv:page:1001:2026-05-27
```

同一用户重复写入不会使估算 UV 按访问次数重复增长。若要统计一周或一个月的去重 UV，可以将每天的 `HyperLogLog` 合并后再统计：

```bash
PFMERGE uv:page:1001:week:2026-W22 \
  uv:page:1001:2026-05-25 \
  uv:page:1001:2026-05-26 \
  uv:page:1001:2026-05-27

PFCOUNT uv:page:1001:week:2026-W22
```

之所以不直接使用 `Set`，是因为 `Set` 必须保存每个去重用户标识，统计精确但内存开销会随用户量持续增长；`HyperLogLog` 只使用固定级别的小内存，特别适合百万级、千万级访问统计和页面、活动、接口等多维 UV 场景。

需要注意的是，`HyperLogLog` 返回的是近似值，不适用于计费、权益发放、精确风控等要求零误差的业务。这类场景应使用 `Set`、位图映射或离线明细去重方案。另外，按天创建 key 时应设置合理过期时间；在 Redis Cluster 中，涉及多个 key 的合并操作还应设计一致的 hash tag，避免跨 slot 问题。

## 核心要点

- 海量 UV 近似统计优先使用 `HyperLogLog`
- `PFADD` 写入用户，`PFCOUNT` 读取 UV
- `PFMERGE` 可统计跨天去重总量
- 单个结构内存最多约 `12 KB`
- 标准误差约为 `0.81%`
- 精确统计场景不能直接依赖它

## 面试追问

### 1. HyperLogLog 与 Set 统计 UV 有什么区别？

`Set` 会保存每个用户标识，可通过 `SCARD` 得到精确 UV，也支持判断具体用户是否访问过，但用户规模越大，占用内存越多。`HyperLogLog` 不保存完整明细，只返回近似去重数量，不能用于可靠查询某个用户是否出现过，但内存成本非常低。

### 2. 如何统计每日 UV 和连续七天累计 UV？

每日 UV 可以按日期建立独立 key，访问时通过 `PFADD uv:2026-05-27 userId` 写入，使用 `PFCOUNT` 查询当天结果。连续七天累计 UV 不能直接相加，因为用户可能跨天重复访问，应通过 `PFMERGE` 合并七天的数据后再执行 `PFCOUNT`。

### 3. 什么情况下不能使用 `HyperLogLog` 统计用户数量？

当结果必须完全精确，或后续需要知道具体有哪些用户、判断某用户是否访问过、支持删除单个用户记录时，`HyperLogLog` 就不合适。例如优惠资格结算、计费、抽奖名单生成等场景，应选择可保留明细且能够精确去重的数据方案。

## 常见误区

**误区：** 将每天的 UV 数量直接相加，就能得到一周累计 UV。  
**正确理解：** 同一用户可能多天访问，直接相加会重复计数；应通过 `PFMERGE` 合并每天的 `HyperLogLog` 后统计去重结果。

**误区：** `HyperLogLog` 既能统计 UV，也能准确判断某个用户是否访问过。  
**正确理解：** `HyperLogLog` 主要用于近似基数统计，不保存完整元素明细；需要准确判断用户是否出现过时，应使用 `Set`、`Bitmap` 或其他明细结构。
