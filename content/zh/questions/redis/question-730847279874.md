## 答案

Redis 的 Lua 脚本功能，本质上是：**让多条 Redis 命令以“原子性”的方式一次执行。**

Lua 脚本会在 Redis 服务端执行，执行期间不会被其他客户端命令打断，因此非常适合：

* 分布式锁
* 库存扣减
* 秒杀系统
* 限流
* Redis 事务增强

Redis 使用 EVAL 命令执行 Lua 脚本。

Lua 脚本最大的特点：

* 原子性 ： Lua 脚本的所有命令在执行过程中是原子的，避免了并发修改带来的问题。
* 减少网络开销 ： 通过在服务器端执行脚本，减少了客户端和服务器之前的网络往返次数，提高性能。
* 支持复杂业务逻辑 ： 可以批量更新、条件更新、超过单个 Redis 命令能力。
* 提高并发性能 ：天然原子性、减少网络 IO、服务端执行（避免了逻辑频繁切换）

---

**Lua 脚本基本语法** ：

Redis 执行 Lua，使用 EVAL ，例如 ：`EVAL “return redis.call(‘set’,‘name’,‘tian’)” 0`
含义：
* 执行 Lua 脚本
* redis.call 调用 Redis 命令
* 0 表示没有 key 参数

**Redis Lua 常用参数** ：

`EVAL script numkeys key [key …] arg [arg …]`
比如：`EVAL “return redis.call(‘get’,KEYS[1])” 1 name`，这里 `KEYS[1] = name` 。

ARGV：表示普通参数。例如：`EVAL “return redis.call(‘set’,KEYS[1],ARGV[1])” 1 name tian` 等价于：`SET name tian`

**扩展知识：**

真实业务场景：  秒杀库存扣减（高频）

普通写法：先查库存（GET stock） 再扣减 （DECR stock） 这样会有并发问题。

Lua 写法：Redis 内部一次执行判断库存，扣减库存 -- 不会超卖 


```lua
if stock > 0 then
decr
return success
else
return fail
end
```
  
 Lua 缺点： Lua 执行期间会阻塞 Redis ，因为 Redis 是单线程，Lua 太复杂会导致整个 Redis 卡住，所以 Lua 必须短小，切勿大循环（for）。
