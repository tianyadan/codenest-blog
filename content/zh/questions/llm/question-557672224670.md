## 答案

## 简短答案

混合检索（Hybrid Search）是指：
> 同时结合「关键词检索（Keyword Search）」和「向量检索（Vector Search）」两种检索方式，共同召回知识库内容，再进行融合排序的检索方案。
其核心目标是：

```text
既保证关键词匹配能力
又保证语义理解能力
```
解决 RAG 中：
```text
关键词检索找不到语义相近内容
向量检索找不到精确关键词内容
```
的问题。

目前企业级 RAG 项目几乎都会采用：
```text
混合检索 + Rerank
```
方案。

---

# 什么是混合检索？
传统 RAG 流程：
```text
用户问题
    ↓
Embedding
    ↓
向量检索
    ↓
TopK召回
```
混合检索流程：
```text
用户问题
       ↓
 ┌────────────┐
 │            │
 ▼            ▼
关键词检索   向量检索
 │            │
 └─────┬──────┘
       ▼
    结果融合
       ▼
     Rerank
       ▼
       LLM
```
本质：
```text
Keyword Search
+
Vector Search
=
Hybrid Search
```
---
# 什么是关键词检索？
关键词检索也叫：
```text
全文检索
倒排索引检索
```
例如：

用户提问： Nacos配置中心怎么实现？



知识库： SpringCloud Nacos配置中心原理



由于：
```text
Nacos
配置中心
```
关键词完全匹配。
因此：
```text
ElasticSearch
Lucene
BM25
```
可以快速命中。

---
## 优势
```text
精准匹配
速度快
```
例如：
```text
订单号
用户名
接口名称
异常码
IP地址
TraceId
```
效果极好。

---
## 缺点
无法理解语义。
例如：
用户提问：
```text
服务发现如何实现？
```
文档内容：
```text
Nacos注册中心工作机制
```
没有：
```text
服务发现
```
这个词。
关键词检索可能找不到。

---
# 什么是向量检索？
向量检索通过 Embedding 模型：
```text
文本
 ↓
向量
```
然后计算语义相似度。
例如：
问题：
```text
服务发现如何实现？
```
文档：
```text
Nacos注册中心工作机制
```
虽然词不同。
但是语义接近。
向量检索仍然能找到。

---
## 优势
理解语义。
例如：
```text
Redis是什么
Redis作用是什么
Redis能干什么
```
虽然写法不同。
Embedding 后语义接近。

---
## 缺点
精确匹配能力弱。
例如：
用户提问：
```text
错误码 ERR-50001 是什么？
```
Embedding 很可能认为：
```text
ERR-50001
ERR-50002
ERR-50003
```
都差不多。
导致召回错误。

---
# 为什么需要混合检索？
因为：
```text
关键词检索有盲区
向量检索也有盲区
```
两者互补。

---
## 场景一：专业术语
用户提问：
```text
RocketMQ 的 NameServer 是什么？
```
关键词：
```text
NameServer
```
非常关键。
必须精准匹配。
适合：
```text
BM25
```
---
## 场景二：自然语言问题
用户提问：
```text
微服务如何找到其它服务？
```
知识库：
```text
Nacos服务发现机制
```
关键词不同。
只能依赖：
```text
Embedding
```
理解语义。

---
# 混合检索如何实现？
## 方案一：双路召回
同时执行：
```text
BM25
+
Vector Search
```
例如：
关键词检索：
```text
Top10
```
向量检索：
```text
Top10
```
最终：
```text
Top20
```
合并。
然后：
```text
Rerank
```
排序。

---
## 方案二：加权融合
例如：
```text
最终得分
=
关键词得分 × 40%
+
向量得分 × 60%
```
公式：
```text
FinalScore
=
0.4 × BM25Score
+
0.6 × VectorScore
```
根据总分排序。

# 面试回答
混合检索（Hybrid Search）是指同时结合关键词检索和向量检索两种检索方式进行知识召回的技术方案。
其中关键词检索基于 BM25、倒排索引等算法，擅长精确匹配专业术语、编号、接口名等内容；向量检索基于 Embedding 和向量相似度计算，擅长理解语义相近的问题。
在 RAG 场景下，单独使用关键词检索容易漏掉语义相关内容，而单独使用向量检索又容易丢失精确关键词匹配能力。因此企业级 RAG 系统通常采用混合检索，通过双路召回、结果融合以及 Rerank 重排序，提高知识召回率和最终回答质量。

# 面试官可能的追问

## 追问1：混合检索和 Rerank 有什么区别？

### 答案
混合检索解决：
```text
召回问题
```
目标：
```text
尽可能找全相关文档
```
Rerank 解决：
```text
排序问题
```
目标：
```text
从召回结果中选出最相关内容
```
流程：
```text
Hybrid Search
      ↓
Recall
      ↓
Rerank
      ↓
TopK
```
---

## 追问2：为什么不能只用向量检索？
### 答案
因为向量检索擅长语义匹配。
但对于：
```text
错误码
订单号
用户名
接口名
TraceId
```
等精确内容效果较差。
因此企业场景通常需要结合关键词检索。

---

## 追问3：混合检索为什么能提升 RAG 效果？
### 答案
因为：
```text
关键词检索负责精准匹配
向量检索负责语义匹配
```
二者互补。
这样既能召回：
```text
完全相同关键词
```
又能召回：
```text
语义相近内容
```
从而提升召回率和答案准确率，减少大模型幻觉。
