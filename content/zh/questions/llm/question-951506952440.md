## 答案

## 简要回答

提示压缩（Prompt Compression）是指在不明显损失关键信息的前提下，减少发送给大模型的上下文内容。在 RAG 中使用提示压缩，主要是为了节省 Token、降低成本，并提高检索内容的有效利用率。

## 专业名词解释

### RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种先从知识库检索相关内容，再将检索结果与用户问题一起发送给大模型生成答案的技术方案。

### Token

Token 是大模型处理文本时的基本计量单位。模型输入和输出都会消耗 Token，Token 数量越多，成本越高，上下文窗口占用也越大。

### 上下文窗口

上下文窗口是模型一次能够处理的最大输入长度。如果输入内容超过窗口限制，部分信息可能被截断或无法参与推理。

## 面试回答

提示压缩本质上是一种上下文优化技术，其目标是在保留核心语义的前提下，尽可能减少发送给大模型的内容长度。

在 RAG 系统中，知识库检索通常会返回多个文档分段。如果把所有检索结果原样拼接到 Prompt 中，容易出现三个问题：第一是 Token 消耗过高，增加推理成本；第二是超出模型上下文窗口限制；第三是大量无关或重复信息会干扰模型理解，导致回答质量下降。

因此，RAG 系统通常会在检索之后增加提示压缩环节，例如删除重复内容、提取关键句、过滤低相关内容、生成摘要或者重排序后只保留最重要的内容。这样既能减少 Token 消耗，又能让模型聚焦于真正有价值的信息，从而提高回答质量和响应速度。

简单来说，RAG 负责“找信息”，而提示压缩负责“筛信息”，两者通常配合使用。

## 核心要点

- 提示压缩用于减少上下文长度
- 核心目标是降低 Token 消耗
- 可避免超出模型上下文窗口
- 能减少无关信息干扰
- 常用于 RAG 检索结果优化

## 面试追问

### 1. 提示压缩和摘要生成有什么区别？

摘要生成是提示压缩的一种实现方式，侧重将长文本概括成短文本。而提示压缩范围更广，还包括去重、过滤、关键词提取、内容裁剪、重排序等多种优化手段。

### 2. RAG 中不做提示压缩会有什么问题？

检索结果过多时，可能导致 Token 成本上升、上下文窗口溢出、响应变慢。同时大量低相关内容会稀释关键信息，影响模型对问题的理解和最终回答质量。

## 常见误区

**误区：** 提示压缩就是简单截断文本。  
**正确理解：** 提示压缩的核心是保留关键信息，而不是机械删除内容。高质量压缩通常需要结合相关性分析、摘要提取或重排序策略。

**误区：** 只要模型上下文足够大，就不需要提示压缩。  
**正确理解：** 即使拥有超长上下文窗口，过多无关内容仍会增加成本并干扰模型推理，因此提示压缩依然具有价值。
