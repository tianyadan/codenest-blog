## 简短答案

Google ADK（Agent Development Kit）是 Google 推出的智能体开发框架，用于帮助开发者快速构建、编排、部署和管理 AI Agent（智能体）。
它提供了 Agent、Tool、Workflow、多 Agent 协作等能力，可以理解为 Google 在 Agent 领域对标 LangGraph、CrewAI、AutoGen 的解决方案。
一句话概括：

**ADK 是 Google 官方推出的 AI Agent 开发框架，用于构建生产级多智能体应用。**

---

# 什么是 Google ADK？
ADK 全称：
**Agent Development Kit**
即：
**智能体开发工具包**
Google 推出 ADK 的目的，是解决当前 Agent 开发过程中存在的问题：
- Agent 逻辑分散
- Tool 调用混乱
- 多 Agent 协作复杂
- 工作流难维护
- 缺乏统一标准
因此 Google 提供了一套完整框架，让开发者能够像开发 SpringBoot 项目一样开发 Agent。
可以把它理解为：
 
| 领域 | 框架 |
|---|---|
| Java Web | SpringBoot |
| Python Web | Django |
| AI Agent | Google ADK |

# ADK 的核心能力有哪些？
Google ADK 主要提供五大能力：

### 1、Agent 定义
开发者可以定义自己的 Agent。
例如：
```text
客服Agent
运维Agent
面试Agent
招聘Agent
代码生成Agent
```
每个 Agent 都可以拥有自己的：
- Prompt
- Memory
- Tool
- Workflow
- 推理能力
例如：
```python
support_agent = Agent(
    name="SupportAgent",
    instruction="你是一名客服专家"
)
```
---

### 2、Tool 调用
Agent 本身不具备访问外部世界的能力。
因此需要 Tool。
例如：
- 数据库查询
- 调用接口
- 搜索引擎
- 发送邮件
- 调用 MCP 服务
ADK 可以直接注册工具。
例如：
```python
@tool
def get_weather(city):
    return "晴天"
```
Agent 便能够自主决定是否调用工具。
这与 Spring AI 中的 Function Calling 非常类似。

---
### 3、多 Agent 协作
这是 ADK 最大的特点之一。
例如：
用户提出：
```text
帮我分析今年公司财务情况，并给出招聘建议。
```
一个 Agent 很难完成。
于是：
```text
财务Agent
      ↓
分析财报
招聘Agent
      ↓
制定招聘计划
HR Agent
      ↓
生成最终方案
```
多个 Agent 协同完成任务。
这种模式称为：

**Multi-Agent System（多智能体系统）**

也是当前 Agent 领域最热门的发展方向之一。

---

### 4、Workflow 工作流编排
ADK 支持定义执行流程。
例如：
```text
用户提问
    ↓
知识库检索
    ↓
Rerank
    ↓
调用Agent
    ↓
生成答案
```
或者：
```text
需求分析
    ↓
代码生成
    ↓
代码检查
    ↓
部署验证
```
整个流程可以进行统一管理。
这一点与 Dify Workflow、LangGraph StateGraph 非常相似。

---
### 5、Memory 记忆能力
Agent 可以保存上下文。
例如：
用户第一次提问：
```text
我叫张三
```
第二次提问：
```text
我叫什么？
```
Agent 可以记住：
```text
张三
```
这就是 Memory。
ADK 支持：
- 短期记忆
- 长期记忆
- 会话记忆
- 外部存储记忆

# ADK 和 MCP 的关系是什么？
很多面试官喜欢问这个问题。
两者关系如下：
| 技术 | 作用 |
|---|---|
| MCP | Agent 调用工具标准 |
| ADK | Agent 开发框架 |

可以理解为：

```text
ADK 负责开发 Agent
MCP 负责连接工具
```
例如：
```text
ADK Agent
      ↓
MCP Client
      ↓
数据库
      ↓
搜索引擎
      ↓
企业系统
```
因此：

**ADK 和 MCP 并不是竞争关系，而是协作关系。**

# ADK 和 LangGraph 的区别
| 对比项 | Google ADK | LangGraph |
|---|---|---|
| 出品方 | Google | LangChain |
| 开发语言 | Python | Python |
| 核心定位 | Agent 开发框架 | Agent 工作流框架 |
| 多 Agent | 支持 | 支持 |
| Workflow | 支持 | 强项 |
| Google 生态 | 深度集成 | 一般 |
| Gemini 集成 | 原生支持 | 需额外配置 |

目前来看：
- Google 生态优先选择 ADK
- LangChain 生态优先选择 LangGraph

# 面试回答
Google ADK（Agent Development Kit）是 Google 推出的智能体开发框架，用于构建、编排和管理 AI Agent。它提供 Agent、Tool、Memory、Workflow 和 Multi-Agent 等核心能力，帮助开发者快速构建生产级智能体系统。

ADK 可以理解为 AI 时代的 SpringBoot，它统一了 Agent 的开发模式，并支持多个 Agent 协同工作。同时 ADK 能够与 MCP、Gemini、大模型工具调用等能力结合，实现复杂业务流程自动化，因此被广泛应用于智能客服、知识库问答、智能运维和企业 AI Agent 系统建设。

# 面试官追问

### 追问1：ADK 和 Dify 有什么区别？

**答案：**
Dify 更偏向低代码平台，通过拖拽工作流即可构建 AI 应用，适合业务人员和快速验证场景。
ADK 属于开发框架，需要编写代码，但灵活性更高，更适合复杂业务系统和生产环境。
简单理解：
- Dify = AI 版低代码平台
- ADK = AI 版 SpringBoot

---
### 追问2：ADK 和 MCP 是什么关系？
**答案：**
ADK 负责开发 Agent。
MCP 负责让 Agent 调用外部工具。
两者关系类似：

```text
SpringBoot
     +
MySQL Driver
```
ADK 构建业务逻辑，MCP 提供标准化工具接入能力。

---
### 追问3：为什么企业会需要多 Agent？
**答案：**
因为现实业务往往涉及多个领域知识。
例如生成一份招聘计划可能同时涉及：
- 财务分析
- 人力资源
- 法务审核
- 数据统计
如果全部交给一个 Agent，Prompt 会非常复杂。
因此企业更倾向于采用多个专业 Agent 分工协作，每个 Agent 只负责自己的领域，从而提高准确率、可维护性和扩展性。
