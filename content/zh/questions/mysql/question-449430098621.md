## 答案

**无标准答案**，但有建立索引原则：

1 ）按需建索引，不要预防性建立索引，先跑业务，等慢查询出来了再针对性建立索引。

2 ）优先考虑联合索引。一个联合索引能覆盖多个查询场景，比建多个单列索引划算。联合索引 `idx_a_b_c` 可以支持 `where a = ？`、`where a = ？ b = ？`、`where a = ？ and b = ？and c = ?` 三种查询。

3 ）单表索引数量一般控制在 5-6 个以内。阿里巴巴的 Java 开发手册建议单表索引不要超过 5 个，联合索引字段不要超过 5 个。

4 ） 对于写多读少的表更要慎重建索引。日志表、消息表这种写入量大的场景索引越少越好，甚至只留主键。 可以用主从备份，主库只负责写，从库只负责读，代价是可能会有数据延迟。

5 ）定期清理无用索引。 可以用 `sys.schema_unused_indexes` 查看那些索引长期没有用过，该删就删。
