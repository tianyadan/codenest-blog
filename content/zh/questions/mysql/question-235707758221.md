## 答案

**索引不一定有效** 。 建了索引不代表查询就会用，用了索引也不代表查询就会快。

MySQL 最终是否走索引，**靠的是优化器的成本计算**。优化器会评估走走索引和全表扫描各自的 I/O 成本和 CPU 成本，哪个便宜用哪个。有时候全表扫描成本确实很低，比如表就几百行数据，走索引还不如直接全表扫描；有时候是统计信息不准，导致优化器算错账，明明该走索引却选了全表扫描。

排查索引有效最直接的办法就是用 **EXPLAIN** 命令。在 SQL 前面加上 EXPLAIN ，就能看到 MySQL 选择的执行计划。重点看：

1 ) <span style="color:#e74c3c">type</span>（访问类型） : ALL （全表扫描），range（范围扫描），ref（等值匹配），index（全索引扫描）。ALL 基本就是没走索引，ref 和 range 才是真正利用了索引的快速查找能力。

2 ）<span style="color:#e74c3c">key</span>（实际使用的索引名称）: NULL 代表没用上索引。

3 ）<span style="color:#e74c3c">rows</span> （预估扫描的行数）: 这个数字越大说明查询代价越高。
