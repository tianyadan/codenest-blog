## 答案

1 ） **高频查询优先** ： 分析业务里哪些查询执行频率最高，针对这些查询设计覆盖查询。

2 ）**不要贪多**：覆盖索引不是把所有列都塞进去。索引列越多，索引越大，写入维护成本越高。要权衡读写比例。

3 ）**利用联合索引** ：把 where 条件的列和 select 的列放到一个联合索引里。比如经常执行select a,b from t where c = 1, 可以建(c,a,b) 索引。

4 ) **主键天然在索引里**。二级索引的叶子节点会自动存主键，所以 select 的列如果包含主键，不用额外把主键加到索引里。
