## 答案

并不是建立的索引都能派上用场 ，以下场景会导致索引失效。

1 ） **对索引列使用函数** 。 `where year (create_time) = 2026` 用不上 create_time 的索引 。 改成 `where create_time >= '2026-01-01' and create_time < '2027-01-01' `

2 ) **隐式类型转换** 。 字段是 varchar 但传入数字 ，MySQL 会做隐式类型转换 ，索引失效。 比如 phone 是 varchar ， `where phone = 16655558889` 会失效，改成  `where phone = '16655558889' ` 。

3 ）**左模糊查询**。 `where name like '%张'` 用不上索引，只有右模糊 `where name like '张%'` 能用。

4 ) **or 连接不同列**。 `where a = 1 or b = 2` ,如果 a 和 b 分别有索引，可能都用不上。可以改成 `UNION`

5 ） **联合索引不满足最左匹配** 。索引 (a,b,c) ，查询 where b = 1 用不上 

6 ） **优化器认为全表扫描更快**。 查询返回数据量超过表的 15%-30% ，优化器可能放弃索引 。
