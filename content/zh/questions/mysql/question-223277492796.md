## 答案

MySQL 的**最左前缀匹配原则**：**联合索引从左往右用，不能跳着用。**

假设有一个联合索引：


```sql
INDEX idx_name_age_city (name, age, city)
```
那么 MySQL 使用索引的规则是：

| 查询条件|是否使用索引  | 
| --- | --- |
| WHERE name = 'Tom' | ✅  |
| WHERE name = 'Tom' AND age = 18 | ✅  |
| WHERE name = 'Tom' AND age = 18 AND city = 'BJ' | ✅  |
| WHERE age = 18 |  ❌ |
|WHERE city = 'BJ' | ❌  |
| WHERE name = 'Tom' AND city = 'BJ' | ⚠️ 只用到 name  |

核心规则：
1. 必须从最左列开始匹配 ： 没有 name，后面的 age、city 都白搭
2. 中间不能断： 跳过 age，直接用 city，索引就“断了”
3. 遇到范围查询会停止匹配 ： `WHERE name = 'Tom' AND age > 18 AND city = 'BJ'`
实际只用到：name + age ， city 用不上索引。


**MySQL 联合索引遵循最左前缀原则，从左到右连续匹配，遇到范围查询停止。**

**在实际开发中，设计联合索引时，要把区分度高、使用频率高的字段放在最左边，这样索引利用率更高。**
