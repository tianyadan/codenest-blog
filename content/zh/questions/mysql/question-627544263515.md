## 答案

1 ） **使用覆盖索引**

如果查询需要的列都在索引里，就不需要回表了。比如索引是`(age,name)`,查询select name from user where age = 20, 索引叶子节点存了age,name,主键，直接返回就行。

EXPLAIN 的 Extra 列显示 Using index 就说明走了覆盖索引，没有回表。

2 ） **索引下推**

MySQL 5.6 引入的优化。假设索引是`(age,name)`, 查询 `where age = 20 and name like '% 张'`，以前的做法是先通过 age = 20 拿到所有主键，全部回表拿到完整数据，再从 Server 层启用 name 过滤。

有了索引下推，在<span style="color:#00bcd4">引擎层</span>就能用 `name like '% 张'` 先过滤一遍，不符合条件的直接跳过不回表，大大减少回表次数。

3 ）**减少 select * 使用**

<span style="color:#e74c3c">不要图方便什么都用 Select * </span>, 如果只需要索引里的字段，显示指定列名，避免不必要的回表。
