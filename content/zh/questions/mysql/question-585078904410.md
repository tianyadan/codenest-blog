## 答案

MYSQL 排序数据有两条路 ： 一种是利用**索引天然排序** ，另一种是手动 **filesort（文件排序）** 。 

如果 ORDER BY 的字段正好有索引且排序方向一致，那么 MySQL 就顺着 B+树叶子节点扫就行，无需额外排序，效率极高。

如果没命中索引，或索引顺序不对，就会走 filesort（文件排序） 。

此时 MySQL 会用 `sort_buffer` 这个缓冲区来排序，filesort 排序会根据查询所有字段的大小与`sort_buffer_size` 缓存大小做对比，如果查询字段缓存过多就使用**外部归并排序**，默认的`sort_buffer_size` 大小是(256KB)，可以修改。

**内存排序又分两种：** 划分依据是max_length_for_sort_data。如果查询数据>`max_length_for_sort_data` 就选择双路排序，否则就走单路排序。`max_length_for_sort_data`( 默认 1KB)
- 单路排序：把所有 SELECT 字段都放进 sort_buffer 里面排序，排完直接返回。 （无需回表，一次搞定）
- 双路排序: 只放排序字段和行 ID(row.id)，排好序后返回 Id，再根据 Id再回表取其他字段。（排序完后需要回表，多一次 IO）


**外部归并排序:**
- 数据量超过 `sort_buffer_size`, 就要做外部归并排序，MySQL把数据分割成和sort_buffer_size  差不多的规模大小 的多个小块，在内存使用**快速排序**排好序，然后放到磁盘的外部**临时存储**，然后再使用**归并排序**把小文件组合成大文件。
