## 答案

**批量插入数据库就是把多条数据积攒到一起，用一条 SQL 或一次性数据库交互一口气插进去，不是一条一条慢慢插入。**

## 优势：

- **减少网络往返**：
单条插入每条数据都要在客户端与数据库握手一次，10 万条就是 10 万次 RTT，批量插入一次发过去，网络开销直接降到原来的几百分之一。

- **摊薄数据库开销**：
每条 SQL 执行都要走解析，编译，写日志这套历程，批量操作把这些固定成本分摊到几千万条数据上，单条成本可以忽略不计。

**举个例子**：

单条插入：

```sql
INSERT INTO user(name, age) VALUES ('zhangsan', 18);
INSERT INTO user(name, age) VALUES ('lisi', 19);
..
```

 批量插入:
 
 
```sql
INSERT INTO user(name, age) VALUES
('zhangsan', 18),
('lisi', 19),
...
```
