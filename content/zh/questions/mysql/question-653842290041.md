## 答案

**回表** 不只是多查一次那么简单，还会带来大量的<span style="color:#e74c3c">随机 I/O</span>。

因为通过二级索引拿到的主键 Id 不是连续的。比如 age = 20 的人很多，在 age 索引里按年龄排序，但对应的 id 是无序的，可能是2、58、199、1024 这些散落分布，拿到这些 id 去聚簇索引查，每次都要定位到不同的数据页，产生大量的随机 I/O 。 

顺序 I/O 可以利用磁盘的预读，一次读一大块连续数据，非常快。随机 I/O 每次都要随机寻址，磁盘得来回转。慢的多。所以如果查询返回几万行数据，每行都要回表，性能会急剧下降。
