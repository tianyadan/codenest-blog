## 答案

GC 调优（Garbage Collection Tuning，垃圾回收调优）简单说就是：
👉 **让 JVM 更高效地回收内存、减少停顿时间、提高系统吞吐量**。

**GC 是干嘛的**？

- GC（垃圾回收）是 JVM 自动帮你清理“没用了的对象”的机制。
比如一个对象再也没被引用了，它占着堆内存就没意义，GC 就会帮你干掉它。
这听起来很美好——但是 GC 是有“代价”的：
当它在清理时，可能会让程序“卡”一下（Stop The World）。
GC 调优，就是尽量让这种卡顿更少、更短。

**调优目标？**

-  减少 Full GC 次数（Full GC 一次，全世界都停）
-  降低 GC 停顿时间（Stop The World 越短越好）
-  提高吞吐量（更多时间花在业务逻辑上，而不是 GC）
-  控制内存使用（防止频繁 GC 或 OOM）

**常见手段？**

- 通过 JVM调整堆大小

```
-Xms2g -Xmx2g
```

- 优化新生代、老年代比例

```
-XX:NewRatio=3
```

- 选择合适的 GC 收集器

| 名称 | 优势 |
| --- | --- |
| Serial | 单线程、简单 单核小内存应用 |
| Parallel |  吞吐量优先 批处理、后台计算 |
| CMS | 低延迟、老牌，对延迟敏感的系统 |
| G1 | 平衡型、高级，大多数现代系统推荐|
| ZGC / Shenandoah | 超低延迟，对延迟极端敏感的高端应用|

大多数情况下，现在直接选 G1 就够香了。
新项目一般都用 G1 或 ZGC。

- 调整 GC 触发条件

```
-XX:MaxGCPauseMillis=200
```
告诉 JVM：“兄弟，我希望每次 GC 停顿别超过 200ms。”
