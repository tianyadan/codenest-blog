## 答案

**==：**

* 基本类型：比较值


```java
int a = 10;

int b = 10;

System.out.println(a == b); // true
```

* 引用类型：比较两个变量里存的对象地址是否一样：



```java
User u1 = new User("张三");

User u2 = new User("张三");

System.out.println(u1 == u2); // false
```
虽然内容都是张三，但它们是两个不同对象，可以理解为：u1 -> User对象A ，u2 -> User对象B ，地址不同，所以 == 是 false。



**equals()：**

equals() 是 Object 类里的方法。

* 默认情况下，它和 == 一样，也是比较地址。

```java
public boolean equals(Object obj) {

    return (this == obj);

}
```

所以如果一个类没有重写 equals()，那它比较的还是地址。

 ### 了解更多：
 

**1 ) 为什么 String 用 equals 比较内容？**

因为 String 重写了 equals() 方法。


```java
String s1 = new String("abc");

String s2 = new String("abc");

System.out.println(s1 == s2);      // false

System.out.println(s1.equals(s2)); // true
```

解释：s1 和 s2 是两个不同对象，所以 == 为 false ， 但它们的字符串内容都是 abc，所以 equals 为 true 。


**2 )String 特殊情况：字符串常量池**


```java
String s1 = "abc";

String s2 = "abc";

System.out.println(s1 == s2);      // true

System.out.println(s1.equals(s2)); // true
```

这里 s1 == s2 是 true，因为字符串字面量 "abc" 会进入字符串常量池 , 两个变量指向的是同一个常量池对象。

s1 -> 常量池里的 "abc" , s2 -> 常量池里的 "abc"

**3 )包装类要小心:**


```java
Integer a = 100;

Integer b = 100;

System.out.println(a == b); // true
```

```java
Integer c = 200;

Integer d = 200;

System.out.println(c == d); // false
```

原因是 Integer 默认缓存了 -128 ~ 127 范围内的对象 ,所以100 使用缓存对象，地址一样,200 创建新对象，地址不同 .包装类比较数值，别用 ==，用 equals()。


```java
Integer x = 200;

Integer y = 200;

System.out.println(x.equals(y)); // true
```


**4 )自定义对象怎么比较内容？**

如果你希望两个对象只要字段一样就算相等，就要重写 equals()。


```java
class User {

    private String name;

    public User(String name) {

        this.name = name;

    }

}
```

默认情况：

```java
User u1 = new User("张三");

User u2 = new User("张三");

System.out.println(u1.equals(u2)); // false
```

因为没重写 equals()，比较的是地址。

重写后：


```java
class User {

    private String name;

    public User(String name) {

        this.name = name;

    }

    @Override

    public boolean equals(Object obj) {

        if (this == obj) return true;

        if (obj == null || getClass() != obj.getClass()) return false;

        User user = (User) obj;

        return java.util.Objects.equals(this.name, user.name);

    }

}
```

再比较：

```java
User u1 = new User("张三");

User u2 = new User("张三");

System.out.println(u1.equals(u2)); // true
```

**5 ) 重写 equals 通常也要重写 hashCode**

这个很重要，面试必问 , 如果两个对象 equals() 返回 true，那么它们的 hashCode() 必须相同。

否则在 HashMap、HashSet 里会出问题。


```java
Set<User> set = new HashSet<>();

set.add(new User("张三"));

set.add(new User("张三"));
```

如果只重写 equals()，不重写 hashCode()，可能会导致两个“相等”的对象都被放进去。

标准写法：


```java
@Override

public int hashCode() {

    return java.util.Objects.hash(name);

}
```
