## 答案

**Exception**和 **Error** 都是 **Throwable** 的子类，在 java 中代码只有继承了 Throwable 类的示例才可以被 throw 或者被 catch，都是表示在程序运行时发生的异常状况或错误情况。

Exception 表示**可以被处理**的程序异常，Error 代表系统级**不可恢复**的错误。


Exception 常见的子类有：IOException、SQLException、NullPointerException、IndexOutOfBoundsException 等。

Exceotion 又分为 Checked Exception（编辑期异常）和 Unchecked Exception（运行时异常）。Checked Exception 异常需要显示处理，例如使用 try...catch... 或通过 throw 声明抛出。

Error 代表严重错误，通常是 JVM 层次内系统级错误，无法预料的错误，例如内存耗尽:OutMemoryError、栈溢出:StackOverflowError 。

处理异常时需要注意的六个点：
1. 尽量不要捕获类似与 Exception 这样的通用异常，而应该捕获特定的异常。
2. 不要'吞'了异常不处理。
3. 不要延迟处理异常。
4. try...catch...范围越好越好。
5. 不要通过异常来控制程序流程
6. 不要再 finally 代码快中处理返回值或直接 return
