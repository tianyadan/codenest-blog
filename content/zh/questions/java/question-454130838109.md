## 答案

**Java 只有值传递。**

基本类型传的是值副本；引用类型传的是“对象地址的副本”。所以方法里可以修改对象内容，但不能让外部引用指向新对象。

但容易混淆的是：
**对象作为参数传递时，传递的是“对象引用的副本”，不是对象本身，也不是引用本身。**


```java
void change(User u) {
    u.setName("张三"); // 外部对象会变
    u = new User();   // 外部引用不会变
}
```

### 扩充知识：

**1 ） 什么是值传递？**

> 值传递: 方法调用时，把实参的值复制一份，传给形参。形参怎么改，都不会影响原来的实参变量。

例如：


```java
public static void change(int x) {
    x = 100;
}

public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a); // 10
}
```

这里 a 的值是 10，传给 x 的时候复制了一份。

所以：x = 100; 只是改了 x，不会影响 a。

---

**2 ) 什么是引用传递？**

> 引用传递是： 方法调用时，把变量本身的引用地址传过去，形参和实参指向的是同一个变量。

如果 Java 支持引用传递，那么方法里重新赋值，会影响外面的变量。

比如理论上的引用传递效果应该是：

change(obj);

```java
change(obj);
```

方法里：`obj = new Object();`

外面的 obj 也会被改成新对象 , **但 Java 不会这样**。

---

**3 ) Java 对象传参到底传的是什么？**

Java 对象变量里存的不是对象本身，而是对象的引用地址。

例如：`User user = new User("张三");`

可以理解为：user 变量里存着一个地址：0x001 , 0x001 指向堆里的 User 对象 .

调用方法时：change(user); **Java 会把 user 变量里的地址值复制一份给形参** 

也就是：main 方法里的 user = 0x001 , change 方法里的 u  = 0x001

两个变量都拿着同一个地址，所以都能访问同一个对象。
