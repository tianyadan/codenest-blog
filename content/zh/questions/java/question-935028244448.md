## 答案

1. **final** ： 用于修饰类、方法和变量，主要用来设计不可变类、确保类的安全性、优化性能（编译器优化）。

特别解释：

- 类：被 final 修饰的类不能被继承
- 方法：被 final 修饰的方法不能被重写。
- 变量：被 final 修饰的变量不可被重新赋值，常用语定义常量。

2. **finally**：与 try-catch 语句块结合使用，用于确保无论是否发生异常，finally 代码快内代码都会执行。

主要用于释放资源（如关闭文件、数据库连接等）以保证及时发生异常，资源也能被正确释放。

3. **finalize()**: 是 Object 类中的方法，允许对象在被垃圾回收前进行清理操作。

较少使用，通常用于回收非内存资源（如关闭文件或释放外部资源），但不建议依赖于它，因为 JVM 不保证 finalize()方法会被及时执行。

**在 JDK9 之后 finalize()方法被废弃，因为 Java 提供了更好的替代方案 如：AutoCloseable 接口和 try-with-resource 语句。**
