## 答案

## 先说答案 
**有 5 种事务隔离级别：**

1. DEFFAULT（默认）:使用底层数据库的默认隔离级别。如果没有数据库特定的设置，通常会默认为 READ_COMMITTED

2. READ_UNCOMMITTED( 读未提交）：最低的隔离级别，允许事务读取尚未提交的数据，可能会导致脏读、不可重复读、幻读。

3. READ_COMMITTED(读已提交)：仅允许读取已经提交的数据，避免了脏读，但可能会导致出现不可重复读和幻读问题。

4. PERATABLE_READ（可重复读）：确保在同一个事务内的多次读取结果一致，避免脏读和不可重复读。但可能会有幻读的问题。

5. SERIALIZABLE（可串行化）：最高的隔离级别，通过强制事务按顺序执行，完全避免脏读、不可重复读、幻读，代价是性能明显下降。


### 概念名词解释：
- 脏读：一个事务读取了 另一个尚未提交的事务数据，如果该事务回滚会导致数据不一致。
- 不可重复读：在同一事务内多次读取，前后数据不一致，因为其他事务修改了该数据并提交。
- 幻读：在一个事务内的多次查询，查询结果集不同，因为其他事务插入或删除了数据。


### Spring 中事务隔离级别的设置：


```
@Transactional(isolation = Isolation.REPEATABLE_READ)
public void processTransaction() {
    // 事务逻辑
}
```

可以通过 `isolation` 属性，开发者可以灵活地指定每个事务的隔离级别。

可以根据不同需求权衡合适的事务隔离级别：
- 要求性能高、数据一致性不高场景 推荐低隔离级别（**读未提交**、**读已提交**）
- 要求数据一致性高场景 推荐**可重复读**和**串行化**
