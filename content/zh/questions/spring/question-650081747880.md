## 答案

## **🏷 先说一说什么是作用域(Scope)：**

一句话：是 Spring 容器里，Bean 的“**存活范围**”和“**使用方式**”

详细说：
- 这个 Bean 是整个应用只创建一个？
- 还是每次请求都创建一个新的？
- 还是每个线程单独一份？

这些都是由**作用域**决定的。


## **🔑 常见的作用域有哪些？**

在 Spring/Spring Boot 里，有 5 种常用作用域（其中 2 种常用，3 种主要用于 Web 环境）：

1. **singleton**（单例）,默认作用域。容器里只创建一个 Bean 实例，全局共享。
- 举例:声明一个 @Service，Spring 启动时只会造一个对象，所有地方注入的都是同一个。
- 场景：大多数无状态服务
2. **prototype**（原型，多例）
- 每次从容器 getBean() 时，都会创建一个新的实例。
-  举例：你要一个“笔”的 Bean，每次拿都是一支新笔。
-  场景: 有状态对象（比如临时计算对象）。
3. **request**（请求作用域，Web 环境）
- 每个 HTTP 请求创建一个 Bean，请求结束后销毁。
- 举例: 用户 A 发请求 → 得到 Bean1；用户 B 发请求 → 得到 Bean2。
4. session（会话作用域，Web 环境）
- 每个用户 Session 拥有一个 Bean，不同用户之间不共享。用户 A 登录 → 创建 Bean1（整个 session 内复用）；用户 B 登录 → 创建 Bean2。
- 场景：用户会话级别的数据存储。
5. application（应用作用域，Web 环境）
- 整个 Web 应用（ServletContext 范围）只有一个 Bean，全局共享。
- 类似于 singleton，但范围是 ServletContext，而不是 Spring IOC 容器。
- 场景：应用全局共享的资源。


⚡ 一句话总结：
	•	作用域 = Bean 在容器里的生存范围。
	•	默认是 singleton（全局单例），
	•	需要“临时对象”就用 prototype，
	•	Web 开发里还会用 request / session / application 作用域。
