## 答案

要想解决循环依赖问题，最关键的就是**提前暴露未完全创建的 Bean**。

spring 中主要是用**三级缓存**来解决循环依赖：

1. 一级缓存(Singleton Objects Map)用于存储完全初始化完成的单例 Bean
2.  二级缓存(Early Singleton Objects Map) 用于存储**尚未完全初始化**，但**已经实例化**的 Bean，用于提前暴露对象，避免循环依赖问题。
3.  三级缓存(Singleton Factories Map)用于存储对象工厂，当需要时，可以通过工厂创建早期 Bean，尤其是针对为了支持 AOP 代理对象的创建。

步骤：

1. Spring 首先创建 Bean 实例，并将其加入三级缓存中。
2. 当一个 Bean 依赖另一个为初始化的 Bean 时，Spring 会从三级缓存中获取 Bean 的工厂，并生成该 Bean 的对象（若有代理对象就生成代理对象）
3. 代理对象存入二级缓存，解决循环依赖。
4. 一旦所有依赖 Bean 被完全初始化，Bean 将转移到一级缓存中。

举例：

我有两个类，A 需要 B，B 又需要 A，👉 这就是循环依赖。

```
@Component
public class A {
    @Autowired
    private B b;
}

@Component
public class B {
    @Autowired
    private A a;
}
```

**📌 如果没有三级缓存会怎样？**
	•	Spring 创建 A，发现 A 需要 B → 去创建 B
	•	创建 B 时又发现 B 需要 A → 但是 A 还没创建完！
	•	这时候就死锁了：两个都要等对方，没法继续。
        
**📌 三级缓存是怎么解决的？**
Spring 有三个缓存：
1. 一级缓存（singletonObjects）：存放完全初始化好的单例对象
2. 二级缓存（earlySingletonObjects）：存放半成品对象（可能带代理）
3. 三级缓存（singletonFactories）：存放对象工厂（用来生成对象或代理）
        
        
        
**🔄 流程举例（一步步走）**
1. Spring 开始创建 A, A 还没初始化好，但 Spring 先放一个 A 的工厂 到 三级缓存。
2. Spring 发现 A 需要 B,去创建 B。
3. Spring 开始创建 B,同理，先放一个 B 的工厂 到 三级缓存,发现 B 需要 A。
4. B 需要 A,Spring 先看一级缓存：A 不在（还没完全好）。再看二级缓存：也没有（代理/半成品还没生成）。最后看三级缓存：找到了 A 的工厂！Spring 就用这个工厂生成了一个 A 的代理对象（或半成品 A），放进 二级缓存。然后把这个 A（半成品）注入到 B。👉 这样，B 就创建成功了。
5. B 创建完毕，放入一级缓存
6. 回到 A,现在 B 已经准备好了，可以顺利注入到 A。A 完成初始化，放入一级缓存。
7. 最后,所有对象都在 一级缓存 里，正常使用
