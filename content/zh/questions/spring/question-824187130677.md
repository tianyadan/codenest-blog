## 答案

**✨ 概括重点**

它是 Spring 中 基于经典的 MVC 模式(Model-View-Controller)来开发的 web 应用模块。Spring MVC 将请求处理分为三层：**模型层（Model）、视图层（ View）、控制层（controller）**，提供一种松耦合的方式将用户请求、业务逻辑处理、视图渲染分离。

**📎 主要功能**

1. 请求映射
2. 数据绑定
3. 表单处理
4. 异常处理


**🎯 工作流程**
1. **客户端请求**： 浏览器向服务端发送 Http 请求。
2. **DospatcherServlet**：所有请求都会先由核心前端控制器 `DispatcherServlet` 接收，它充当整个流程的调度中心。
3. **处理器映射**：DospatcherServlet根据请求的 URL 使用处理器映射找到对应的控制器(Controller)
4. **控制器**：控制器接收请求并处理业务逻辑，通常通过注解`@Controller` 和 `@RequestMapping` 定义请求的映射方法。
5. **模型和视图**：控制器处理完业务逻辑后，将数据封装到模型对象中，并指定返回的视图名称。
6. **视图解析器**：DospatcherServlet调用视图解析器，将逻辑视图名称解析成为实际的视图，如 JSP、Thymeleaf 等模版引擎。
7. **视图渲染**：视图渲染引擎根据模型中的数据生成 HTML 页面并返回给客户端。
