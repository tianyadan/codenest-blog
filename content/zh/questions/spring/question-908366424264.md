## 答案

**@Transactional 失效的常见原因**

1. 内部方法调用（最经典）
    • 事务基于代理对象，调用必须走代理。
    • 如果在同一个类中用 this.xxx() 调用另一个 @Transactional 方法，就绕过了代理，AOP 没有介入 → 事务失效。
    
2. 方法不是 public 。
   •	Spring AOP 默认只拦截 public 方法。
   •	protected、private、package-private 的方法上加事务不会生效。
   
3. 异常没有抛出 / 抛出类型不对
   •	默认只会对 运行时异常（RuntimeException, Error） 回滚。
   •	如果 try...catch 把异常吞了，或者抛的是 CheckedException（比如 IOException），就不会触发回滚。
   
4. 类没有被 spring 管理
   • 如果 new 了一个对象（自己实例化），那 Spring 根本没机会生成代理，自然也就没有事务。
   
**解决方法：**

1. 通过代理调用

1️⃣ 注入自身（代理对象）

```
@Autowired
private UserService self; 
self.txMethod();
```
 
2️⃣ 使用 AopContext.currentProxy()


```
((UserService) AopContext.currentProxy()).txMethod();
```

3️⃣ 把方法拆出去放到另一个 spring Bean 中再去调用。


2. 方法修饰符必须是 public ，要避免 private/protected，保证能被 AOP 拦截。

3. 异常处理正确 

1️⃣ 不要吃掉异常

2️⃣ 如果要回滚 Checked 异常 ，加上： 

```
@Transactional(rollbackFor = Exception.class)
```

4. 确认是 spring 管理的 Bean 

类上要加 @Service、@Component，确保是 Spring 容器生成的对象。
