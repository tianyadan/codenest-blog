## 答案

### 简介：
FactoryBean 是 Spring 提供的一个**特殊接口**，允许开发者通过**自定义的逻辑创建复杂** Bean，与普通的 Bean 不同，通过 FactoryBean 创建的 Bean **一定不是 FactoryBean 本身**，而是它生产的对象。

核心概念：FactoryBean 实现了一个 F`actoryBean<T>` 接口的 Bean，通过它可以自定义复杂对象的创建逻辑，Spring 容器会调用 getObject() 方法来获取实际的 Bean 实例。

### 举例:


```
    public class CarFactoryBean implements FactoryBean<Car> {
    @Override
    public Car getObject() throws Exception {
        // 自定义复杂逻辑造车
        Car car = new Car();
        car.setBrand("Tesla");
        car.setPrice(300000);
        return car;
    }
        @Override
    public Class<?> getObjectType() {
        return Car.class;
    }
}
    
```
	
  CarFactoryBean 自己是工厂,当你去 Spring 容器里 getBean("carFactoryBean") 时：Spring 给你的是 Car 对象（成品车），而不是 CarFactoryBean 本身（工厂）。
  
### 🤔 那我能不能要工厂本人？
  
 --  能！只要在名字前面加个 **&**,这样才会得到“工厂对象”本身。
 
 
```
CarFactoryBean factory = (CarFactoryBean) context
                        .getBean("&carFactoryBean");
```

### ✅ 为什么要用 FactoryBean?
1. 隐藏复杂的创建过程：
-  有些对象创建很复杂，比如：需要连数据库、远程调用 API、或者依赖很多配置文件。如果你在业务代码里手动 new，那业务就被“对象装配逻辑”污染了。用 FactoryBean → 统一把“造对象”的复杂逻辑封装起来，Spring 帮你管理。👉 业务只关心结果（对象），不关心过程（怎么造出来的）。
2. 实现解耦：业务类只拿到成品 Bean，不需要知道对象是用 new 出来的，还是通过框架造的。这就避免了直接依赖“具体实现类”，符合 **面向接口编程** 的思想。
3. 动态灵活：FactoryBean 可以根据环境变量、配置文件，甚至运行时条件，来决定生产哪个对象。


### 📦 举个更生活化的例子：

假设你要一台 电脑：
- 普通 Bean：Spring 直接给你 new Computer()。
- FactoryBean：Spring 给你的是“电脑厂商的销售点”，你只管拿电脑，至于背后是用组装机、品牌机、还是高配游戏机，Spring 自己决定。这样你作为消费者，就完全不需要直接操作电脑零件（对象创建细节），拿成品就行。
