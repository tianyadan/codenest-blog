## 答案

**父容器**：父容器指的是 Spring 的根容器，通常是 Spring 应用上下文（ApplicationContext），如ContextLoaderListener 加载的根容器。它主要用于管理应用程序全局的 Bean，如服务层（Service）、数据访问层（DAO）等。

**子容器**：是 Web 容器，每个 DispatcherServlet 实例都会创建一个子容器，用于管理 Web 层（如控制器和拦截器）中的 Bean。


**父子容器关系：**
- **子容器可以访问父容器的 Bean**：如果一个 Bean 在父容器中定义，子容器也可以直接访问它。这种机制有助于 web 层（子容器）使用服务层或 DAO 层（父容器）中的 Bean。

- **父容器不能访问子容器的 Bean**：父容器无法访问子容器中的 Bean，这是 Spring MVC 的设计之一。父容器的 Bean 和子容器中的 Bean 被分开管理。，避免了不必要的耦合。
