## 答案

ObjectFactory 是 Spring 框架中的一个接口，主要用于**延迟获得 Bean 实例**。

**🛠 不想写太官方的，来点纯粹的：**

- 它是接口，定义在 Spring 里，功能超级简单：**按需(延迟)** 获取一个 Bean 实例。


```
public interface ObjectFactory<T> {
    T getObject() throws BeansException;
}
```

它就只有一个方法：getObject()，返回一个对象。

**📦 它和 BeanFactory 的区别：**
- BeanFactory：是整个 IOC 容器，负责管理所有 Bean。**BeanFactory：大管家（仓库管理员）。**
- ObjectFactory：只是一个“小工具接口”，帮你延迟获取某个 Bean。**ObjectFactory：小帮手（代取快递的人）。**


**🤔 那它有啥用？**
- **解决循环依赖**:如果 A 依赖 B，B 又依赖 A，直接注入会死循环。Spring 会用 ObjectFactory 做一个“代理引用”，先占个位，等需要时再真正取 Bean。
- **懒加载**（Lazy Init）, 某些 Bean 很重，启动时不想马上创建。用 ObjectFactory 包装一下 → 只有你真的 getObject() 时才会去 BeanFactory 拿这个 Bean。


**📖 举个简单例子：**
```@Component
public class OrderService {
    // 不直接注入 Bean，而是注入一个 ObjectFactory
    @Autowired
    private ObjectFactory<PaymentService> paymentServiceFactory;

    public void createOrder() {
        // 只有用到的时候才真正去 BeanFactory 拿 PaymentService
        PaymentService payment = paymentServiceFactory.getObject();
        payment.pay();
    }
}
```
这样 PaymentService 就不会在容器启动时立即创建，而是等调用 createOrder() 时才生成。
