## 答案

SpringBoot 支持的嵌入式 Web 容器主要有**三种 Servlet 容器**：Tomcat（默认）、Jetty、Undertow。
如果使用响应式编程（WebFlux），SpringBoot **还支持基于 Reactor 的 Netty 作为容器**。
开发者可以通过排除默认依赖并引入相应的 starter 来切换容器。

## 1. Servlet 容器（传统 Web 应用）

(1) Tomcat（默认）
- 在引入 spring-boot-starter-web 时，自动依赖了 spring-boot-starter-tomcat。
- 优点：**成熟稳定**，生态广泛。
- 缺点：并发性能相对 Jetty/Undertow 差一些。

(2) Jetty
- 替代 Tomcat 的一种选择。
- 特点：轻量、适合长连接、**广泛用于高并发场景**。
- 要用 Jetty，只需要排除 Tomcat 依赖，引入：

```
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>
```

(3) Undertow
- JBoss 社区维护的容器，性能很强，支持非阻塞 IO。
- 特点：**高性能、低内存消耗、支持 WebSocket**，适合微服务场景。
- 配置方法同样是排除 Tomcat，引入：


```
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-undertow</artifactId>
</dependency>
```

## 2. Reactive 容器（响应式 Web 应用）

- SpringBoot 2.x 开始支持 WebFlux，这是基于 Reactor 的响应式编程模型。
- 默认使用 Netty 作为嵌入式容器。
- 特点：非阻塞、高并发、低延迟，适合处理流式数据和高 QPS API


```
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```
