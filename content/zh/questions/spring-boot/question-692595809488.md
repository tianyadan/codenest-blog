## 答案

两者的区别就在于书写格式，对于配置而言效果是一样的。
- application.properties 是键值对配置，键和值之间用等号或冒号分割。

```
server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/test
spring.datasource.username=root
```

- application.yml 是使用YAML 格式，具有更好的层级结构，适合复杂配置，阅读性更加。


```
server:
  port: 8080
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/test
    username: root
```

**使用场景**
- 推荐用 .yml：
配置多层级、结构复杂的应用（比如微服务、多个数据源、Redis 集群等）。
- 用 .properties：
简单项目、配置量少，或者对 YAML 不熟悉的团队。

优先级加载顺序：
application.properties > application.yml 如果同时配置一样的参数，最终生效的是application.properties 中配置的参数信息。
