## 答案

主要有五点特性：

1）**自动配置**： 这是最核心的特性，它能够根据项目里引入的依赖（比如引入了`spring-boot-starter-web`，就知道是 web 项目），自动配置 Tomcat、Spring MVC 等组件。底层是通过`@EnableAutoConfiguration+spring.factories` 实现的条件装配机制，节省了大量的 XML 配置。

2）Starters 依赖启动器：把常用场景的依赖打包成 starters（比如 web 对应 spring-boot-starter-web、数据库对应 spring-boot-starter-data-jpa）项目里只需要引入一个 starter，就能自动拉取该场景下所有需要的依赖，避免手动管理依赖版本冲突。

3）内嵌容器：内置 Tomcat、Jetty、Undertow 等 Web 容器，可以打包成 jar 包直接通过 java -jar 命令启动，不要额外部署 WAR 包，简化部署流程。

4）外部化配置：支持多种配置环境（如application.yml /application-dev.yml ）。并可以通过环境变量、命令行参数等方式覆盖，实现应用与配置解耦。

5）Actuator（监控与管理）：提供了一系列用于监控和管理生产级别应用的端点（Endpoints），让我们能轻松查看应用健康状态、指标、日志等信息，提供应用的可观测性与生产级监控能力。

**总的来说：**

SpringBoot 通过 **Starters 依赖启动器**简化了依赖管理，通过**自动装配**简化配置，**内嵌容器**简化部署，通过**外部化配置**实现应用与配置解耦，并通过 **Actuator** 提供应用的可观测性与生产级监控能力。
