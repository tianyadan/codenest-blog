## 答案

### 1.启动 Main() 方法

应用从 main（）方法启动，并通过 SpringApplication.run()引导应用启动；

### 2.创建 SpringApplication

应用创建 SpringApplication 对象，推断应用类型，设置初始化器、启动监听器、确定主应用类。

### 3.准备环境

在启动过程中准备应用环境，加载配置文件、系统环境变量以及命令行参数。

### 4.创建并刷新 ApplicationContext

 创建应用上下文，加载配置类和自动配置类，注册 Bean 并执行依赖注入等初始化操作。
 
### 5.创建上下文时启动嵌入式 web 服务器
 
 对于 web 应用，SpringBoot 会自动启动嵌入式 web 容器（如 Tomcat）并注册相关的 servlet和 Filter。
 
### 6.发布应用已启动事件
  
  对应监听 stated 事件逻辑会被触发。
  
### 7.执行 commandLineRunner 和 ApplicationRunner

 在应用启动完成后，执行实现了 commandLineRunner 和 ApplicationRunner 接口的初始化逻辑。
 
### 8.发布 ready 事件、应用启动完成

触发 ApplicationReadyEvent，应用进入运行状态，处理业务请求或任务。



### 针对No.6 补充：
发布应用已启动事件：

- 你可以把 Spring Boot 应用 想象成一台汽车。当你按下启动键（应用开始启动）时，Spring 会在不同阶段“广播消息”，告诉所有对这个阶段感兴趣的“听众”（监听器）：
- 👉“喂，我已经启动到这个阶段啦！”
- stated 事件就是其中一个“广播信号”。如果你写了一个事件监听器来监听这个事件，那么它就会在这个时候被触发。

- 就像电梯开门时会“叮”一声，提醒人们可以进来了。这就是事件发布 → 监听器响应。


### 针对No.7 补充：
执行 CommandLineRunner 和 ApplicationRunner：

这两个接口是 Spring Boot 提供的“钩子”，让你在应用启动完成后插入自己的初始化逻辑。
- CommandLineRunner：会把启动时传进来的参数（args）直接丢给你用（字符串数组形式）。
- ApplicationRunner：和上面类似，只是参数会被封装成一个 ApplicationArguments 对象，操作更方便一些。

它们都会在 Spring Boot 应用 启动完成后，正式运行前 执行。
👉 也就是说，如果你有一些初始化工作（比如加载缓存、检查配置、预热数据），可以放在这里写。

✅ 类比：
- 车子（应用）已经启动好了。
- 现在你可以选择：
- CommandLineRunner：直接拿到一袋原始食材（字符串数组），自己处理。
- ApplicationRunner：拿到一份整理好的食材清单（ApplicationArguments），更直观。


**示例代码：**

```
import org.springframework.boot.CommandLineRunner;
import org.springframework.boot.ApplicationArguments;
import org.springframework.boot.ApplicationRunner;
import org.springframework.stereotype.Component;

// CommandLineRunner 示例
@Component
public class MyCommandLineRunner implements CommandLineRunner {
    @Override
    public void run(String... args) throws Exception {
        System.out.println(">>> CommandLineRunner 执行了，参数如下：");
        for (String arg : args) {
            System.out.println("参数: " + arg);
        }
    }
}

// ApplicationRunner 示例
@Component
public class MyApplicationRunner implements ApplicationRunner {
    @Override
    public void run(ApplicationArguments args) throws Exception {
        System.out.println(">>> ApplicationRunner 执行了，参数如下：");
        System.out.println("原始参数: " + args.getSourceArgs().length);
        System.out.println("是否包含 --debug 参数: " + args.containsOption("debug"));
    }
}
```

**输出结果：**
```
>>> CommandLineRunner 执行了，参数如下：
参数: --debug
参数: --name=tian

>>> ApplicationRunner 执行了，参数如下：
原始参数: 2
是否包含 --debug 参数: true
```
