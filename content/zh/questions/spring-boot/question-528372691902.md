## 答案

## 核心要点：
1. **实现 CommandLineRunner 接口**
2. **实现 ApplicationRunner 接口**
3. **使用@PostConstruct 注解**
4. **使用 InitializingBean 接口**
5. **使用 Spring 事件监听器**
6. **自定义 BeanFactoryPostProcessor 和 BeanPostProcessor**

## 实现：

1. **实现 CommandLineRunner 接口**
在 SpringBoot 启动完成后（ApplicationContext 加载完毕），会执行 run 方法。
```
import org.springframework.boot.CommandLineRunner;
import org.springframework.stereotype.Component;

@Component
public class MyCommandLineRunner implements CommandLineRunner {
    @Override
    public void run(String... args) {
        System.out.println("【CommandLineRunner】项目启动完成后执行，参数：" + String.join(",", args));
    }
}
```
2. **实现 ApplicationRunner**
和 CommandLineRunner 类似，但参数更强大，封装成了 ApplicationArguments。

```
import org.springframework.boot.ApplicationArguments;
import org.springframework.boot.ApplicationRunner;
import org.springframework.stereotype.Component;

@Component
public class MyApplicationRunner implements ApplicationRunner {
    @Override
    public void run(ApplicationArguments args) {
        System.out.println("【ApplicationRunner】项目启动完成后执行，参数：" + args.getOptionNames());
    }
}
```

3. **使用@PostConstruct**
在 Bean 初始化完成后（依赖注入后）会执行此方法，早于 CommandLineRunner。

```
import jakarta.annotation.PostConstruct;
import org.springframework.stereotype.Component;

@Component
public class MyPostConstruct {
    @PostConstruct
    public void init() {
        System.out.println("【@PostConstruct】在 Bean 初始化后执行");
    }
}

```

4. **使用 InitializingBean 接口**
和 @PostConstruct 类似，但写法不同。Spring 在属性注入完成后调用。
```
import org.springframework.beans.factory.InitializingBean;
import org.springframework.stereotype.Component;

@Component
public class MyInitializingBean implements InitializingBean {
    @Override
    public void afterPropertiesSet() {
        System.out.println("【InitializingBean】在 Bean 属性设置完成后执行");
    }
}
```

5. **使用 Spring 事件监听器**
可以监听 Spring 内置事件，比如 ApplicationReadyEvent（项目完全启动后）。
```
import org.springframework.boot.context.event.ApplicationReadyEvent;
import org.springframework.context.ApplicationListener;
import org.springframework.stereotype.Component;

@Component
public class MyApplicationReadyListener implements ApplicationListener<ApplicationReadyEvent> {
    @Override
    public void onApplicationEvent(ApplicationReadyEvent event) {
        System.out.println("【ApplicationListener】ApplicationReadyEvent：应用已准备就绪");
    }
}
```

6. **自定义 BeanFactoryPostProcessor 和 BeanPostProcessor**
这两个是 Bean 生命周期的底层钩子。BeanFactoryPostProcessor：在 Bean 定义（BeanDefinition）加载完成后，实例化前执行。BeanPostProcessor：在 Bean 初始化前后执行。

```
import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanFactoryPostProcessor;
import org.springframework.beans.factory.config.BeanPostProcessor;
import org.springframework.beans.factory.config.ConfigurableListableBeanFactory;
import org.springframework.stereotype.Component;

@Component
public class MyBeanFactoryPostProcessor implements BeanFactoryPostProcessor {
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        System.out.println("【BeanFactoryPostProcessor】修改或检查 BeanDefinition");
    }
}

@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        if (beanName.equals("myPostConstruct")) {
            System.out.println("【BeanPostProcessor】在 Bean 初始化之前：" + beanName);
        }
        return bean;
    }
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        if (beanName.equals("myPostConstruct")) {
            System.out.println("【BeanPostProcessor】在 Bean 初始化之后：" + beanName);
        }
        return bean;
    }
}
```
