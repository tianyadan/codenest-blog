## 答案

## 1. 最大连接数（Max Connections）

- 含义：同一时刻，服务器（比如内嵌的 Tomcat）能够维持的 并发 TCP 连接数 上限。
- 举例：
- - 一个浏览器和服务端建立一个 TCP 连接，这就是一个连接。
- - 假设 Tomcat 配置 maxConnections=200，当有第 201 个连接到来时，Tomcat 会 拒绝新连接 或让其等待，直到有空闲连接释放。

**👉 它更偏底层，控制 网络层面的资源。**



## 2. 最大请求数（Max Threads / Max Request Processing）

- 含义：同一时刻，同时能够被 Tomcat 线程池处理的 并发 HTTP 请求数。
- 举例：
- - 一个用户连接建立后，可能会发送多个请求（如加载页面、加载图片、Ajax 请求）。
- - 假设 Tomcat 配置 maxThreads=200，最多只能同时处理 200 个请求，第 201 个请求就会进入队列等待，直到有线程空闲。

**👉 它更偏业务层，控制应用处理能力。**
