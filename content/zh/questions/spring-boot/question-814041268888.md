## 答案

SpringBoot 的自动配置是基于 **`@EnableAutoConfiguration`** 实现的，它通过 AutoConfigurationImportSelector 读取 spring.factories / AutoConfiguration.imports 文件，把对应的 xxxAutoConfiguration 类导入到 Spring 容器。

这些配置类本质上就是普通的 @Configuration 配置类，只是加了很多 @ConditionalOnClass、@ConditionalOnMissingBean、@ConditionalOnProperty 等条件注解，保证只有在合适的条件下才会生效。同时，SpringBoot 会结合 @ConfigurationProperties 读取配置文件中的属性，完成 Bean 的个性化配置。比如，数据源自动配置就是这样完成的。

**执行流程总结**
1. SpringBoot 启动 → 扫描 @SpringBootApplication → 找到 @EnableAutoConfiguration。
2. 通过 AutoConfigurationImportSelector 加载所有自动配置类。
3. 根据条件注解判断哪些配置类需要生效。
4. 最终把符合条件的 Bean 注册到 Spring 容器里。

**补充解释：**
条件注解 (Conditional 系列)
- @ConditionalOnClass：类路径下有这个类才生效。
- @ConditionalOnMissingBean：只有你没有手动配置时，才自动帮你装配。
- @ConditionalOnProperty：根据配置文件里的开关来决定是否生效。
