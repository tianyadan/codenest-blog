## 答案

## 简要回答

Spring Cloud 常用注册中心包括 `Nacos`、`Eureka`、`Consul` 和 `Zookeeper`。目前国内 Spring Cloud Alibaba 项目中，`Nacos` 使用较为广泛，因为它同时支持服务注册发现和配置管理。

## 专业名词解释

### 注册中心

注册中心用于维护服务实例信息。服务启动后将自身地址注册进去，其他服务通过注册中心发现可调用的实例，从而避免硬编码服务地址。

### 服务发现

服务发现是指消费者根据服务名查询可用实例列表，并选择具体实例发起调用的过程。

## 面试回答

Spring Cloud 中常用的注册中心主要有 `Nacos`、`Eureka`、`Consul` 和 `Zookeeper`。

`Eureka` 是 Spring Cloud Netflix 体系中经典的服务注册与发现组件，早期项目使用较多。`Nacos` **属于 Spring Cloud Alibaba 生态，既可以作为注册中心，也可以作为配置中心，功能比较完整**，在国内微服务项目中应用广泛。`Consul` 除了服务发现，还提供健康检查和键值存储能力。`Zookeeper` 本质上是分布式协调组件，也可以用于服务注册与发现，但在新的 Spring Cloud 业务项目中通常不是首选。

实际选型时，如果项目使用 Spring Cloud Alibaba，**通常优先考虑 `Nacos`**；维护存量 Netflix 体系项目时，可能继续使用 `Eureka`；对多语言服务治理、健康检查有需求时，也可以考虑 `Consul`。

## 核心要点

- 常用注册中心有 `Nacos`、`Eureka`、`Consul`、`Zookeeper`
- `Nacos` 同时支持注册中心和配置中心
- `Eureka` 是 Netflix 体系的经典方案
- `Consul` 支持服务发现与健康检查
- Spring Cloud Alibaba 项目通常优先选择 `Nacos`

## 面试追问

### 1. 注册中心在微服务架构中解决了什么问题？

注册中心解决的是服务地址动态管理问题。服务实例启动、扩容、下线时，可以自动注册或注销；服务消费者通过服务名获取可用实例列表，无需在配置中写死每个服务的地址。

### 2. Nacos 和 Eureka 有什么主要区别？

`Eureka` 主要聚焦服务注册与发现，属于 Spring Cloud Netflix 体系中的经典组件；`Nacos` 除了服务注册与发现，还提供配置管理能力，更适合 Spring Cloud Alibaba 体系下同时管理服务和配置的项目。

## 常见误区

**误区：** Spring Cloud 只能使用 `Eureka` 作为注册中心。  
**正确理解：** Spring Cloud 通过服务发现抽象支持多种注册中心，常见实现包括 `Nacos`、`Eureka`、`Consul` 和 `Zookeeper`。
