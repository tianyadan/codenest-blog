# 后端接口开发需要着重考虑的点

设计一个后端接口时，不要只盯着「能不能跑通」。更稳妥的做法是先把关键质量项过一遍，再开始写实现。

## 优先检查清单

- **参数合法性**：入参类型、必填项、取值范围、边界值是否都覆盖。
- **权限校验**：当前用户是否有权访问该资源，是否存在越权读写风险。
- **是否需要缓存**：读多写少、热点查询是否适合加缓存，以及缓存失效策略是否清晰。
- **是否需要审计日志**：关键写操作、敏感查询是否要留下操作人、时间与变更内容。
- **是否需要 MQ**：是否存在异步解耦、削峰填谷、最终一致性等场景。
- **幂等性是否到位**：重复提交、重试、网络抖动时，会不会造成重复写入或状态错乱。
- **是否需要通知**：操作完成后是否要站内信、邮件、Webhook 或其他下游通知。
- **是否需要单元测试**：核心分支、异常路径、权限失败与幂等场景是否有测试兜底。

## 实践建议

可以先把这份清单固化成接口评审模板。每新增一个接口，都按同样顺序过一遍，能明显减少联调阶段才暴露的权限漏洞、重复提交和日志缺失问题。
