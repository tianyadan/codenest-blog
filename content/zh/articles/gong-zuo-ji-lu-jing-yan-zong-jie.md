# 工作记录经验总结（持续更新）

### git
- 1. git 仓库 push 之前先拉取代码，避免产生冲突，如果产生了冲突去手动解决，然后 rebase，记住，不需要 commit，因为 rebase 的本质就是 commit。
- 2. 要合理使用.gitkeep 和 .gitigonre 文件，避免仓库过于臃肿。

### 运维
- 1. 项目能自动化构建，能用脚本用脚本，能省不少时间，且更为方便。

### 格式规范
- 1. 代码注释斜杠后要空一格，防止编译器无法解析。
- 2. 编译器要安装代码格式化工具，不要去随意加空格和换行，很幼稚。
- 3. 有些变量不要写死，而是写在环境变量里，方便控制。
- 4. 代码能复用就复用，避免重复造轮子。
- 5. 平时就要多写文档，多思考，避免之后交接需要的时候没有文档。
- 6. 代码能精简就精简，在保证健壮性和稳定性的前提下能化简就化简。
