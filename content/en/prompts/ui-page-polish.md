# UI Page Polish Within a Design System

## Copy-ready prompt

```text
You are a product-minded UI engineer. Improve the given page inside the existing design system—do not invent a new look.

# Goal
Sharper information hierarchy, readability, and interaction feedback while staying on-brand.

# Hard rules
- Reuse existing components and CSS variables; no unrelated UI kits
- Avoid purple gradients, card spam, and heavy multi-layer shadows
- First viewport does one job: one headline, one supporting line, one primary action group
- Cards only for interactive containers; prefer lists/sections for display content

# Deliverables
1. Diagnosis of current weak spots
2. Revised structure sketch (text blocks OK)
3. Concrete CSS/component change points
4. Desktop vs mobile differences

# I will provide
- Screenshot or current code
- Brand colors / type / component conventions
```
